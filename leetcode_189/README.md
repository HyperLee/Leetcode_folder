# LeetCode 189 - Rotate Array

這個專案使用 C# 實作 LeetCode 189「Rotate Array」，並提供兩種解法：

- 解法一：三次反轉，符合題目要求的 in-place 思路
- 解法二：額外陣列，將每個元素搬到右移後的位置

主程式會直接執行 5 組固定測資，比對兩種解法的實際輸出與預期答案，方便在本機快速驗證。

## 題目說明

- 題目連結：[LeetCode 189. Rotate Array](https://leetcode.com/problems/rotate-array/description/)
- 題意重述：給定整數陣列 `nums` 與非負整數 `k`，請將陣列中的元素整體向右輪轉 `k` 步。

例如：

- `nums = [1,2,3,4,5,6,7]`，`k = 3`
- 輪轉後結果為 `[5,6,7,1,2,3,4]`

## 限制條件

題目頁面常見約束如下：

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `0 <= k <= 10^5`

## 解題概念與出發點

這題的核心在於：真正有效的位移量不是 `k` 本身，而是 `k % nums.Length`。

原因很直觀：

- 若陣列長度是 `7`，右移 `7` 次後會回到原位
- 右移 `8` 次等同右移 `1` 次
- 右移 `14` 次等同右移 `0` 次

因此，不論採用哪一種解法，第一步都應該先把 `k` 正規化成 `k % n`。

接著就會出現兩條典型思路：

- 若優先考慮題目要求的 in-place 解法，可以使用「整體反轉 + 區間反轉」
- 若優先考慮直觀性，可以建立一個新陣列，直接把元素搬到新位置

這個專案同時保留這兩種思考方式，並在 `Main` 中對照驗證。

## 解法一：三次反轉

### 設計想法

如果我們要把陣列向右移動 `k` 格，原本在尾端的 `k` 個元素，最終會移到前面。

以 `nums = [1,2,3,4,5,6,7]`、`k = 3` 為例：

- 原本尾端的 `[5,6,7]` 會變成新陣列的前段
- 原本前段的 `[1,2,3,4]` 會往後移

若直接逐一搬移元素，實作容易變得繁瑣。反轉法的巧妙之處在於：

1. 先把整個陣列反轉
2. 再把前 `k` 個元素反轉回正確順序
3. 最後把剩餘的 `n - k` 個元素反轉回正確順序

這樣就能在原地完成輪轉。

### 為什麼可行

整體反轉後，原本尾端要移到最前面的元素，會先被集中到前段；只是它們的相對順序顛倒了。

接著再把前 `k` 個元素反轉一次，就能把這一段恢復成正確順序。

剩下那一大段也是同樣道理：它們被翻到了後段，但順序反了，所以再反轉一次即可。

### 範例演示流程

以 `nums = [1,2,3,4,5,6,7]`、`k = 3` 為例：

| 步驟 | 陣列狀態 | 說明 |
| --- | --- | --- |
| 原始資料 | `[1,2,3,4,5,6,7]` | 目標是向右移動 3 步 |
| 全部反轉 | `[7,6,5,4,3,2,1]` | 原本尾端的元素先被集中到前段 |
| 反轉前 3 個元素 | `[5,6,7,4,3,2,1]` | 把新的前段恢復成正確順序 |
| 反轉後 4 個元素 | `[5,6,7,1,2,3,4]` | 把新的後段恢復成正確順序 |

### 演算法步驟

1. 若陣列長度小於等於 `1`，直接返回
2. 計算 `distance = k % nums.Length`
3. 若 `distance == 0`，表示輪轉後不會改變，直接返回
4. 反轉整個陣列
5. 反轉索引區間 `[0, distance - 1]`
6. 反轉索引區間 `[distance, nums.Length - 1]`

### 複雜度分析

- 時間複雜度：`O(n)`
- 空間複雜度：`O(1)`

### 適用情境

這是題目的主推解法，因為：

- 不需要額外陣列
- 滿足 in-place 的設計目標
- 複雜度漂亮，且在面試中很常見

## 解法二：額外陣列

### 設計想法

第二種解法更直觀：既然每個元素右移後都會落到一個新位置，那就直接建立一個新陣列，把每個值放到它應該去的位置。

對索引 `i` 的元素來說，右移 `k` 步後的新位置就是：

`(i + k) % n`

完成後，再把新陣列整體複製回原本的 `nums`。

### 範例演示流程

同樣使用 `nums = [1,2,3,4,5,6,7]`、`k = 3`：

| 原索引 `i` | 原值 | 新索引 `(i + 3) % 7` | 放置後的新陣列狀態 |
| --- | --- | --- | --- |
| `0` | `1` | `3` | `[_ , _ , _ , 1, _ , _ , _ ]` |
| `1` | `2` | `4` | `[_ , _ , _ , 1, 2, _ , _ ]` |
| `2` | `3` | `5` | `[_ , _ , _ , 1, 2, 3, _ ]` |
| `3` | `4` | `6` | `[_ , _ , _ , 1, 2, 3, 4]` |
| `4` | `5` | `0` | `[5, _ , _ , 1, 2, 3, 4]` |
| `5` | `6` | `1` | `[5, 6, _ , 1, 2, 3, 4]` |
| `6` | `7` | `2` | `[5, 6, 7, 1, 2, 3, 4]` |

### 演算法步驟

1. 若陣列長度小於等於 `1`，直接返回
2. 計算 `distance = k % nums.Length`
3. 若 `distance == 0`，直接返回
4. 建立一個長度為 `n` 的新陣列
5. 對每個索引 `i`，將 `nums[i]` 放到 `newArr[(i + distance) % n]`
6. 將 `newArr` 複製回 `nums`

### 複雜度分析

- 時間複雜度：`O(n)`
- 空間複雜度：`O(n)`

### 適用情境

這種方式的優點是：

- 容易理解
- 索引映射很直接
- 很適合初次接觸這題時建立直覺

缺點則是需要額外空間，因此不如翻轉法符合題目進階要求。

## 兩種解法比較

| 項目 | 解法一：三次反轉 | 解法二：額外陣列 |
| --- | --- | --- |
| 是否原地修改 | 是 | 最後會覆寫原陣列，但需要暫存新陣列 |
| 額外空間 | `O(1)` | `O(n)` |
| 理解門檻 | 中等，需要理解反轉技巧 | 低，索引搬移直觀 |
| 面試表現 | 很好，常被視為標準解 | 可作為先想到的基礎解 |
| 推薦程度 | 主推 | 輔助理解與對照 |
