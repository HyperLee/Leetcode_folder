## 題目描述

給定一個大小為 `m x n` 的二維整數網格 `grid` 以及一個整數 `x`。每一次操作，你可以對網格中**任意一個元素加上 `x` 或減去 `x`**。

所謂 **單值網格 (uni-value grid)** 是指網格內所有元素皆相等的網格。

請回傳將 `grid` 變成單值網格所需的**最少操作次數**；若無法達成，請回傳 `-1`。

**約束條件**

- $1 \le m, n \le 10^5$，$1 \le m \cdot n \le 10^5$
- $1 \le x, \text{grid}[i][j] \le 10^4$

---

## 範例

| # | grid | x | 輸出 | 說明 |
|---|------|---|------|------|
| 1 | `[[2,4],[6,8]]` | `2` | `4` | 全部變成 `6`，操作 `(2→6)+(4→6)+(8→6)` 共 `2+1+1 = 4` 次 |
| 2 | `[[1,5],[2,3]]` | `1` | `5` | 全部變成 `3`（中位數），總絕對差 `2+1+0+2 = 5` |
| 3 | `[[1,2,4],[6,8,10]]` | `2` | `-1` | `1 mod 2 = 1`，`2 mod 2 = 0`，模數不同，無解 |

## 解題概念與出發點

### 直覺：操作的「不變量 (invariant)」

每次操作只能 **加上或減去 `x`**，所以對任意元素 $v$、任意整數 $k$ 次操作之後：

$$
(v + k \cdot x) \bmod x \;=\; v \bmod x
$$

這代表一個關鍵性質：**元素對 `x` 取模的結果，永遠不會被操作改變**。

> [!IMPORTANT]
> 兩個元素若要透過加減 `x` 變成相同值，它們對 `x` 取模的結果**必須相同**。否則任憑怎麼操作，都無解。

舉例：當 $x = 2$，偶數永遠是偶數、奇數永遠是奇數，奇偶不可能相遇。

### 直覺：把人聚在一起，往「中間」最快

> 操場上有一群人，要把他們聚集到同一個位置，往哪裡走總距離最短？答案是**中位數**位置。

這就是著名的 **中位數貪心 (Median Greedy)**：對於一維上的點集合，最小化 $\sum |v - t|$ 的最佳目標 $t$ 就是這群點的**中位數**。

## 解法詳解

整體流程分成三個步驟：

```
[判斷是否無解 (mod x 一致性)]
        ↓
[排序後取中位數作為目標值]
        ↓
[計算 ∑|v - median| 並除以 x]
```

### 步驟 1：判斷是否無解

任挑一個元素（例如 `grid[0][0]`）算出 `target = grid[0][0] % x`，然後掃過整個網格：只要存在某個元素 $v$ 使 `v % x != target`，立刻回傳 `-1`。

```csharp
int target = grid[0][0] % x;
foreach (int[] row in grid)
{
    foreach (int num in row)
    {
        if (num % x != target) return -1;
    }
}
```

### 步驟 2：選擇最佳目標值（中位數貪心）

把所有元素攤平成一維陣列 `arr`、排序後取中位數 `median = arr[k / 2]`（其中 `k = m * n`）。

#### 為什麼是中位數？— 公式推導

設目標值為 $t$，定義代價函式：

$$
f(t) = \sum_{v \in \text{grid}} |v - t|
$$

將排序後的元素記為 $a_1 \le a_2 \le \dots \le a_k$。$f(t)$ 是分段線性、凸函數，於是它的最小值出現在「導數從負變正」的點。

對 $t$ 求次梯度：

$$
f'(t) = (\text{ } t \text{ 右邊的點數} ) - ( \text{ } t \text{ 左邊的點數} )
$$

當 $t$ 落在中位數的位置時，左右兩側點數最接近，$f'$ 由負轉正（或為 0），此處即為最小值。

- $k$ 為奇數：唯一最小點為 $a_{\lceil k/2 \rceil}$。
- $k$ 為偶數：在 $[a_{k/2},\; a_{k/2+1}]$ 區間皆可取到最小值。實作上取 `arr[k/2]` 即可（仍落在最佳區間內）。

> [!NOTE]
> 為什麼不用「平均數」？平均數最小化的是 $\sum (v - t)^2$（平方差），而本題每次操作的「成本」是 1（不是 $|v - t|$ 的平方），所以要最小化的是**絕對差總和**，最佳解是**中位數**而非平均數。

### 步驟 3：計算操作次數

由於每次操作只能讓某個元素的值 ±`x`，把 $v$ 變成 `median` 需要的操作次數正是：

$$
\frac{|v - \text{median}|}{x}
$$

而步驟 1 已保證所有 $v$ 滿足 $v \equiv \text{median} \pmod{x}$，故 $|v - \text{median}|$ 必為 $x$ 的倍數，整除一定整齊。

合計所有元素：

$$
\text{answer} \;=\; \sum_{v \in \text{grid}} \frac{|v - \text{median}|}{x} \;=\; \frac{1}{x} \sum_{v \in \text{grid}} |v - \text{median}|
$$

```csharp
Array.Sort(arr);
int median = arr[k / 2];

int operations = 0;
foreach (int num in arr)
{
    operations += Math.Abs(num - median);
}
return operations / x;
```

## 複雜度分析

設 $k = m \times n$。

| 項目 | 複雜度 | 來源 |
|------|--------|------|
| 時間 | $O(k \log k)$ | 排序為主導步驟 |
| 空間 | $O(k)$ | 攤平後的一維陣列 |

## 流程演示

以 `grid = [[1, 5], [2, 3]]`、`x = 1` 為例：

**步驟 1 — 模數一致性檢查**

| 元素 | `v mod 1` |
|------|-----------|
| 1, 5, 2, 3 | 全部為 `0` ✅ |

通過檢查，繼續往下。

**步驟 2 — 排序並取中位數**

```
排序前：[1, 5, 2, 3]
排序後：[1, 2, 3, 5]
              ↑
        median = arr[4/2] = arr[2] = 3
```

**步驟 3 — 累加絕對差並除以 x**

| `v` | `|v - 3|` |
|-----|-----------|
| 1 | 2 |
| 2 | 1 |
| 3 | 0 |
| 5 | 2 |
| **合計** | **5** |

$$
\text{answer} = \frac{5}{1} = 5
$$

對應的操作示例：`1→2→3`（2 次）、`2→3`（1 次）、`3`（0 次）、`5→4→3`（2 次），共 **5 次**。
