# LeetCode 2615 — Sum of Distances（等值距離和）

以 C# (.NET 10) 實作 LeetCode 第 2615 題 **Sum of Distances**，採用「**分組 + 前綴和**」的 $O(n)$ 解法。

- 英文題目：<https://leetcode.com/problems/sum-of-distances/description/>
- 中文題目：<https://leetcode.cn/problems/sum-of-distances/description/>

---

## 題目說明

給定一個 **0-indexed** 整數陣列 `nums`（長度為 $n$），請回傳一個長度相同的陣列 `arr`。對每個下標 $i$，定義同值下標集合

$$
T_i = \{\, j \mid 0 \le j < n,\ j \ne i,\ nums[j] = nums[i] \,\}
$$

則

$$
arr[i] = \sum_{j \in T_i} |i - j|
$$

若 $T_i = \varnothing$（沒有其他同值下標），則 $arr[i] = 0$。

### 範例

| 輸入 `nums`   | 輸出 `arr`      | 說明                                                    |
| ------------- | --------------- | ------------------------------------------------------- |
| `[1,3,1,1,2]` | `[5,0,3,4,0]`   | 下標 0 的 1：`\|0-2\|+\|0-3\| = 5`；下標 3 的 1：`3+1 = 4`  |
| `[0,5,3]`     | `[0,0,0]`       | 每個值皆唯一                                              |

### 限制

- $1 \le \text{nums.length} \le 10^5$
- $0 \le \text{nums}[i] \le 10^9$
- 總和可能很大，回傳型別需為 `long[]`。

---

## 解題概念與出發點

### 直覺解法（會超時）

最直接的做法是雙層迴圈：對每個 `i`，掃描所有 `j` 累加 `|i - j|`，時間複雜度 $O(n^2)$。當 $n = 10^5$ 時顯然無法通過。

### 關鍵觀察

1. **只有同值下標彼此貢獻**。不同值的下標之間完全不影響，因此可把相同值的下標**分成獨立的群組**，每組獨立計算。
2. **按掃描順序收集下標，組內自然遞增**。排序後的組 $a_0 < a_1 < \dots < a_{m-1}$ 讓絕對值 $|a_i - a_j|$ 可以依 $i, j$ 的大小關係拆成兩段（左邊是被減數，右邊是加數），消掉絕對值符號。
3. **拆掉絕對值後只剩線性組合**，可用**前綴和**在 $O(m)$ 時間算完整組答案。

---

## 解法推導

設某個值在 `nums` 中出現的下標組（按原陣列出現順序收集，因此**自動遞增**）為

$$
a_0 < a_1 < \dots < a_{m-1}
$$

> [!IMPORTANT]
> 從這裡開始，下方推導中的 $i, j$ **不再是原題 `nums` 的下標**，而是「**`a` 這個遞增序列裡的位置**」（$0 \le i, j \le m - 1$）。真正要寫入答案的原始下標是 $a_i$。這是為了方便以前綴和處理，才做的變數命名切換。

定義：

- 組內下標總和：$S = \sum_{k=0}^{m-1} a_k$
- 前綴和：$P_i = \sum_{k=0}^{i-1} a_k$（即 $a_i$ **左邊**所有組內下標之和；$P_0 = 0$）

我們要計算的是 $a_i$ 到**組內其他所有位置**的距離總和：

$$
res[a_i] = \sum_{\substack{0 \le j \le m-1 \\ j \ne i}} |a_i - a_j|
$$

由於 $j$ 可能在 $a_i$ 的**左邊**（$j < i$）也可能在**右邊**（$j > i$），兩邊都有貢獻，因此把求和拆成兩半：

$$
res[a_i] = \underbrace{\sum_{j=0}^{i-1} |a_i - a_j|}_{\text{左邊的 } j,\ \text{即 } j < i} + \underbrace{\sum_{j=i+1}^{m-1} |a_i - a_j|}_{\text{右邊的 } j,\ \text{即 } j > i}
$$

> [!NOTE]
> 為什麼會出現 $j < i$？因為我們是站在 $a_i$ 的角度，要累加「所有其他位置到 $a_i$ 的距離」。距離是**雙向**的：比 $a_i$ 早出現的同值下標（$j < i$）要算、比它晚出現的（$j > i$）也要算。這跟原題 $|i - j|$ 本來就對稱（$|i-j| = |j-i|$）相呼應。

因為 $a$ 已遞增，絕對值可以直接去掉：

- 當 $j < i$：$a_j < a_i$，所以 $|a_i - a_j| = a_i - a_j$。
- 當 $j > i$：$a_j > a_i$，所以 $|a_i - a_j| = a_j - a_i$。

於是

$$
res[a_i] = \sum_{j=0}^{i-1}(a_i - a_j) + \sum_{j=i+1}^{m-1}(a_j - a_i)
$$

**左半部**（$j < i$，共 $i$ 項）：$a_i$ 被加了 $i$ 次，$a_j$ 全部被減去一次，剛好是前綴和 $P_i$：

$$
\sum_{j=0}^{i-1}(a_i - a_j) = i \cdot a_i - P_i
$$

**右半部**（$j > i$，共 $m - i - 1$ 項）。利用 $\sum_{j=i+1}^{m-1} a_j = S - P_i - a_i$（總和扣掉左邊的 $P_i$ 與自己 $a_i$）：

$$
\sum_{j=i+1}^{m-1}(a_j - a_i) = (S - P_i - a_i) - (m - i - 1) \cdot a_i
$$

兩者相加化簡：

$$
\boxed{\,res[a_i] = S - 2 P_i + a_i \cdot (2i - m)\,}
$$

### 演算法

1. 用 `Dictionary<int, List<int>>` 把每個值的下標依序收集。
2. 對每一組：
   - 先求 $S$（整組下標總和）。
   - 一邊線性掃描、一邊維護 $P_i$，套用公式直接寫入 `res[a_i]`。

### 複雜度

- **時間**：各組成本 $O(m)$，所有組大小加總為 $n$，總體 $O(n)$。
- **空間**：$O(n)$（雜湊表與輸出陣列）。

> [!NOTE]
> 因為下標最大可達 $10^5$、乘法後仍可能達 $10^{10}$ 級別，務必使用 `long` 累加以避免溢位。

## 範例演示流程

以 `nums = [1,3,1,1,2]` 為例，$n = 5$。

### 步驟 1：分組

依值收集下標：

| 值 | 下標組（已遞增） |
| -- | ---------------- |
| 1  | `[0, 2, 3]`      |
| 3  | `[1]`            |
| 2  | `[4]`            |

### 步驟 2：逐組套公式

**組「值 = 1」：** $a = [0, 2, 3]$、$m = 3$、$S = 0 + 2 + 3 = 5$。

| $i$ | $a_i$ | $P_i$ | $2i - m$ | $res[a_i] = S - 2P_i + a_i(2i - m)$    | 結果 |
| --- | ----- | ----- | -------- | -------------------------------------- | ---- |
| 0   | 0     | 0     | $-3$     | $5 - 0 + 0 \cdot (-3) = 5$             | 5    |
| 1   | 2     | 0     | $-1$     | $5 - 0 + 2 \cdot (-1) = 3$             | 3    |
| 2   | 3     | 2     | $1$      | $5 - 4 + 3 \cdot 1 = 4$                | 4    |

**組「值 = 3」：** $a = [1]$、$m = 1$、$S = 1$，$i = 0$：$1 - 0 + 1 \cdot (0 - 1) = 0$。

**組「值 = 2」：** 同上，只有一項 ⇒ `0`。

### 步驟 3：彙整結果

| 下標 | 0 | 1 | 2 | 3 | 4 |
| ---- | - | - | - | - | - |
| `arr`| 5 | 0 | 3 | 4 | 0 |

最終回傳 `[5, 0, 3, 4, 0]`，與預期一致。
