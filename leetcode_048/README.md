# LeetCode 48. Rotate Image（旋轉影像）

> 給定一個 `n x n` 的二維矩陣，將影像**順時針旋轉 90 度**，且必須**原地（in-place）**修改，不得使用額外矩陣。

## 題目說明

| 項目 | 內容 |
|------|------|
| 題號 | [48. Rotate Image](https://leetcode.com/problems/rotate-image/) |
| 難度 | **Medium** |
| 主題 | Array、Math、Matrix |

### 題意

給定一個 `n x n` 的整數矩陣 `matrix`，將其**順時針旋轉 90 度**。

**限制條件：**
- 必須在原地修改輸入矩陣
- 不得分配額外的二維矩陣

### 範例

**輸入：**
```
1  2  3
4  5  6
7  8  9
```

**輸出：**
```
7  4  1
8  5  2
9  6  3
```

### 旋轉的本質

矩陣順時針旋轉 90 度後，原位置 `(i, j)` 的元素會移動到 `(j, n-1-i)`：

```
原矩陣位置 (i, j)  →  旋轉後位置 (j, n-1-i)
```

---

## 解題思路

直接實作元素從 `(i, j)` 搬到 `(j, n-1-i)` 的原地操作並不容易，因此可以將旋轉**拆解為更簡單的步驟**。兩種主要策略：

| 策略 | 核心概念 | 直覺難度 |
|------|---------|---------|
| 翻轉法 | 上下翻轉 + 對角線轉置 = 旋轉 90° | 容易理解 |
| 循環交換法 | 同時搬動四個對稱位置的元素 | 較為抽象 |

兩者的時間複雜度皆為 **O(n²)**，空間複雜度皆為 **O(1)**。

---

## 解法一：翻轉法（Flip + Transpose）

### 概念

將「順時針旋轉 90 度」分解為兩個獨立操作：

1. **上下翻轉（Vertical Flip）**：第 `i` 行與第 `n-1-i` 行整行交換
2. **對角線轉置（Transpose）**：沿主對角線（左上→右下）交換 `(i, j)` 與 `(j, i)`

> [!NOTE]
> 這兩步操作的組合，數學上等價於位置映射 `(i, j) → (j, n-1-i)`，恰好就是順時針 90 度旋轉。

### 數學推導

```
原始位置:         (i, j)
上下翻轉後:       (n-1-i, j)
再對角線轉置後:   (j, n-1-i)   ← 正是順時針旋轉 90° 的結果
```

### 程式碼

```csharp
public void Rotate(int[][] matrix)
{
    int n = matrix.Length;

    // 步驟 1: 上下翻轉 — 只需遍歷上半部分 (n/2 行)
    for (int i = 0; i < n / 2; i++)
    {
        for (int j = 0; j < n; j++)
        {
            int temp = matrix[i][j];
            matrix[i][j] = matrix[n - 1 - i][j];
            matrix[n - 1 - i][j] = temp;
        }
    }

    // 步驟 2: 對角線轉置 — 只遍歷主對角線右上方
    for (int i = 0; i < n; i++)
    {
        for (int j = i + 1; j < n; j++)
        {
            int temp = matrix[i][j];
            matrix[i][j] = matrix[j][i];
            matrix[j][i] = temp;
        }
    }
}
```

### 為什麼有效？

- **上下翻轉**改變了「行的順序」（列順序不變）
- **對角線轉置**將「行與列互換」
- 兩者組合後，每個元素恰好到達旋轉 90 度的目標位置

## 解法二：四位置循環交換法（Cyclic Swap）

### 概念

直接觀察：旋轉 90 度時，矩陣中每 **4 個對稱位置** 的元素形成一個循環。我們只需遍歷矩陣的 **四分之一區域**，每次同時搬動 4 個元素即可完成旋轉。

四個位置的循環關係：

```
左上 (i, j)          →  右上 (j, n-1-i)
右上 (j, n-1-i)      →  右下 (n-1-i, n-1-j)
右下 (n-1-i, n-1-j)  →  左下 (n-1-j, i)
左下 (n-1-j, i)      →  左上 (i, j)
```

### 遍歷範圍

只需遍歷矩陣的四分之一：

- **行**：`i` 從 `0` 到 `(n+1)/2 - 1`
- **列**：`j` 從 `0` 到 `n/2 - 1`

> [!NOTE]
> `(n+1)/2` 與 `n/2` 的差異確保了奇數矩陣的中心元素不會被重複處理。

### 程式碼

```csharp
public void RotateOptimized(int[][] matrix)
{
    int n = matrix.Length;

    for (int i = 0; i < (n + 1) / 2; i++)
    {
        for (int j = 0; j < n / 2; j++)
        {
            // 暫存左上角的值
            int temp = matrix[i][j];

            // 左上 ← 左下
            matrix[i][j] = matrix[n - 1 - j][i];
            // 左下 ← 右下
            matrix[n - 1 - j][i] = matrix[n - 1 - i][n - 1 - j];
            // 右下 ← 右上
            matrix[n - 1 - i][n - 1 - j] = matrix[j][n - 1 - i];
            // 右上 ← 暫存的左上
            matrix[j][n - 1 - i] = temp;
        }
    }
}
```

## 演示流程

### 範例矩陣

```
1  2  3
4  5  6
7  8  9
```

期望結果（順時針旋轉 90 度）：

```
7  4  1
8  5  2
9  6  3
```

---

### 解法一演示：翻轉法

#### 步驟 1：上下翻轉

將第 0 行與第 2 行交換（第 1 行是中間行，不需要移動）：

```
原始:            上下翻轉後:
1  2  3          7  8  9       ← 原第 2 行
4  5  6    →     4  5  6       ← 不動
7  8  9          1  2  3       ← 原第 0 行
```

交換過程：
| 操作 | 交換位置 | 結果 |
|------|---------|------|
| `swap(matrix[0][0], matrix[2][0])` | `1 ↔ 7` | 完成第 1 列交換 |
| `swap(matrix[0][1], matrix[2][1])` | `2 ↔ 8` | 完成第 2 列交換 |
| `swap(matrix[0][2], matrix[2][2])` | `3 ↔ 9` | 完成第 3 列交換 |

#### 步驟 2：對角線轉置

沿主對角線（左上→右下）交換 `(i, j)` 與 `(j, i)`：

```
上下翻轉後:       對角線轉置後:
7  8  9           7  4  1
4  5  6     →     8  5  2
1  2  3           9  6  3
```

交換過程（只處理主對角線右上方的元素）：
| 操作 | 交換位置 | 說明 |
|------|---------|------|
| `swap(matrix[0][1], matrix[1][0])` | `8 ↔ 4` | `(0,1) ↔ (1,0)` |
| `swap(matrix[0][2], matrix[2][0])` | `9 ↔ 1` | `(0,2) ↔ (2,0)` |
| `swap(matrix[1][2], matrix[2][1])` | `6 ↔ 2` | `(1,2) ↔ (2,1)` |

**最終結果：**
```
7  4  1
8  5  2
9  6  3  ✔ 旋轉完成
```

---

### 解法二演示：四位置循環交換法

遍歷範圍：`i ∈ [0, 1]`、`j ∈ [0, 0]`（只遍歷四分之一區域）

#### 第一輪：`i=0, j=0` — 四個角落交換

```
四個位置:
  左上 (0,0)=1    右上 (0,2)=3
  左下 (2,0)=7    右下 (2,2)=9
```

循環交換流程：
```
temp = 1（暫存左上）

① 左上 ← 左下:  matrix[0][0] = matrix[2][0] = 7
② 左下 ← 右下:  matrix[2][0] = matrix[2][2] = 9
③ 右下 ← 右上:  matrix[2][2] = matrix[0][2] = 3
④ 右上 ← temp:  matrix[0][2] = 1
```

交換後矩陣：
```
7  2  1
4  5  6
9  8  3
```

#### 第二輪：`i=1, j=0` — 四個邊中點交換

```
四個位置:
  左上 (1,0)=4    右上 (0,1)=2
  左下 (2,1)=8    右下 (1,2)=6
```

循環交換流程：
```
temp = 4（暫存左上）

① 左上 ← 左下:  matrix[1][0] = matrix[2][1] = 8
② 左下 ← 右下:  matrix[2][1] = matrix[1][2] = 6
③ 右下 ← 右上:  matrix[1][2] = matrix[0][1] = 2
④ 右上 ← temp:  matrix[0][1] = 4
```

交換後矩陣：
```
7  4  1
8  5  2
9  6  3  ✔ 旋轉完成
```

> [!TIP]
> 中心元素 `5` 位於 `(1,1)`，旋轉後仍在原位，因此不需要處理。這正是遍歷範圍設計（`(n+1)/2` × `n/2`）自動排除中心點的結果。

## 複雜度比較

| | 翻轉法 | 循環交換法 |
|---|--------|-----------|
| 時間複雜度 | O(n²) | O(n²) |
| 空間複雜度 | O(1) | O(1) |
| 遍歷次數 | 兩次（各遍歷約一半矩陣） | 一次（遍歷四分之一矩陣） |
| 理解難度 | 直觀、好記憶 | 較抽象、需想像四個位置 |
| 適合場景 | 面試首選，容易解釋 | 追求一次遍歷的最佳化寫法 |
