# LeetCode 347：前 K 個高頻元素

這是一個以 C# 撰寫的 .NET 10 主控台專案，示範如何從整數陣列中找出出現頻率最高的前 `k` 個元素。專案保留兩種解法：符合 follow-up 時間複雜度要求的桶排序法，以及較直觀、便於對照的字典排序法。

> 題目： [347. Top K Frequent Elements](https://leetcode.com/problems/top-k-frequent-elements/description/)

## 題目說明

給定整數陣列 `nums` 與整數 `k`，回傳出現頻率最高的 `k` 個元素。答案的元素順序可以任意。

例如 `nums = [1, 1, 1, 2, 2, 3]`、`k = 2` 時，數字 `1` 出現 3 次、`2` 出現 2 次、`3` 出現 1 次，因此答案是 `[1, 2]`。

## 限制條件與輸入前提

- `1 <= nums.Length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`
- `1 <= k <= nums` 中相異元素的數量
- 題目保證答案唯一。
- follow-up 要求演算法時間複雜度必須優於 `O(n log n)`，其中 `n` 是陣列長度。

兩個公開方法都依照上述 LeetCode 前提設計，不額外處理 `null`、空陣列或超出相異元素數量的 `k`。在有效輸入下，它們都回傳恰好 `k` 個高頻元素。

## 解題出發點

直接逐一比較每個元素和整個陣列的出現次數，會重複掃描資料。兩種解法都先建立「元素 → 頻率」的 `Dictionary<int, int>`，把統計工作集中在一次走訪完成；差別在於取得前 `k` 名的方式：

| 方法 | 核心做法 | 時間複雜度 | 額外空間 | 是否符合 follow-up |
| --- | --- | --- | --- | --- |
| `TopKFrequent` | 以頻率為索引建立桶，再由高頻向下取值 | `O(n)` | `O(n)` | 是 |
| `TopKFrequent2` | 將頻率表依頻率遞減排序 | `O(n + m log m)` | `O(m)` | 否（最壞為 `O(n log n)`） |

其中 `m` 是相異元素數量，且 `m <= n`。

## 解法一：桶排序法 `TopKFrequent`

### 設計說明

桶排序法的關鍵觀察是：長度為 `n` 的陣列中，任何元素最多只會出現 `n` 次。因此可建立索引範圍從 `0` 到最大頻率的桶陣列；`buckets[f]` 代表「所有恰好出現 `f` 次的元素」。

演算法流程如下：

1. 走訪 `nums`，將每個數字的出現次數記錄到 `counts`。
2. 找出最大頻率，建立對應大小的 `List<int>[] buckets`。
3. 將每個 `(數字, 頻率)` 放入 `buckets[頻率]`。
4. 從最大頻率反向走到 1；每取出一個元素就放進結果，取滿 `k` 個立即停止。

因為桶的索引已經表示頻率，步驟 4 不需要額外排序數字。統計、配置桶、放入桶與反向掃描的總成本都不超過線性範圍，所以最壞時間為 `O(n)`。

### 範例演示

以 `nums = [1, 1, 1, 2, 2, 3]`、`k = 2` 為例：

1. 統計後：`counts = { 1: 3, 2: 2, 3: 1 }`，最大頻率為 3。
2. 依頻率放桶：`buckets[1] = [3]`、`buckets[2] = [2]`、`buckets[3] = [1]`。
3. 由索引 3 向下掃描：先從 `buckets[3]` 取出 `1`，再從 `buckets[2]` 取出 `2`。
4. 已取得兩個元素，立即停止並回傳 `[1, 2]`。

## 解法二：字典排序法 `TopKFrequent2`

### 設計說明

這個方法同樣先用 Dictionary 建立頻率表，但把 `(元素, 頻率)` 配對交給 `OrderByDescending` 依頻率遞減排列。排序後的前 `k` 個鍵就是答案。

演算法流程如下：

1. 走訪 `nums`，建立 `counts`。
2. 將字典項目依 `Value`（頻率）由大到小排序。
3. 依序讀取排序後項目的 `Key`，收集到第 `k` 個元素就停止。

這種寫法易於理解，也很適合先驗證頻率統計邏輯；但排序 `m` 個相異元素需要 `O(m log m)`，當所有元素都不同時會退化為 `O(n log n)`，因此不符合題目 follow-up 的線性時間要求。

### 範例演示

以 `nums = [-1, -1, -1, -2, -2, -3, -3, -3, -3]`、`k = 2` 為例：

1. 統計後：`counts = { -1: 3, -2: 2, -3: 4 }`。
2. 依頻率遞減排序後：`(-3, 4)`、`(-1, 3)`、`(-2, 2)`。
3. 取前兩個項目的鍵，得到 `[-3, -1]`。
