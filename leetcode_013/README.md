## 題目說明

羅馬數字由七種符號組成，每個符號代表一個固定數值：

| 符號 | 數值 |
| --- | ---: |
| `I` | 1 |
| `V` | 5 |
| `X` | 10 |
| `L` | 50 |
| `C` | 100 |
| `D` | 500 |
| `M` | 1000 |

一般情況下，羅馬數字會由大到小、由左到右排列，所以可以直接把每個符號的值相加。例如：

- `III = 1 + 1 + 1 = 3`
- `XII = 10 + 1 + 1 = 12`
- `XXVII = 10 + 10 + 5 + 1 + 1 = 27`

但羅馬數字也存在「減法規則」：當較小的符號放在較大的符號前方時，表示要從右側較大的值中扣掉左側較小的值。題目列出六種常見組合：

| 組合 | 數值 | 說明 |
| --- | ---: | --- |
| `IV` | 4 | `5 - 1` |
| `IX` | 9 | `10 - 1` |
| `XL` | 40 | `50 - 10` |
| `XC` | 90 | `100 - 10` |
| `CD` | 400 | `500 - 100` |
| `CM` | 900 | `1000 - 100` |

題目要求：給定一個有效的羅馬數字字串 `s`，將它轉換為對應的整數。

## 解題概念與出發點

直覺上，若所有符號都由大到小排列，只要從左到右相加即可。但 `IV`、`IX`、`XL` 這類組合會讓較小的符號出現在較大的符號前面，因此不能無條件相加。

關鍵觀察是：

- 如果目前符號的值大於或等於下一個符號，代表是一般加法。
- 如果目前符號的值小於下一個符號，代表目前符號是減法組合的左側符號，應該扣除。

換句話說，每一個符號只需要看「自己」和「右邊下一個符號」的大小關係，就能決定要加還是要減。

以 `MCMXCIV` 為例：

- `M` 後面是 `C`，`1000 >= 100`，所以加 `1000`。
- `C` 後面是 `M`，`100 < 1000`，所以減 `100`。
- `M` 後面是 `X`，`1000 >= 10`，所以加 `1000`。
- `X` 後面是 `C`，`10 < 100`，所以減 `10`。
- `C` 後面是 `I`，`100 >= 1`，所以加 `100`。
- `I` 後面是 `V`，`1 < 5`，所以減 `1`。
- `V` 是最後一個符號，直接加 `5`。

最後得到 `1000 - 100 + 1000 - 10 + 100 - 1 + 5 = 1994`。

## 解法說明

專案目前提供兩個方法，兩者都使用同一個核心策略：建立符號到整數的對應表，然後由左到右掃描字串。

### 解法一：方法內建立對應表

`RomanToInt(string s)` 在方法內宣告 `Dictionary<char, int>`：

```csharp
Dictionary<char, int> dic = new Dictionary<char, int>
{
    {'I', 1},
    {'V', 5},
    {'X', 10},
    {'L', 50},
    {'C', 100},
    {'D', 500},
    {'M', 1000}
};
```

接著逐一讀取 `s[i]`：

1. 先取得目前符號的值 `value`。
2. 如果 `i` 已經是最後一個位置，直接加上 `value`。
3. 如果右邊下一個符號的值小於或等於目前值，代表一般加法，直接加上 `value`。
4. 如果右邊下一個符號的值大於目前值，代表減法組合，改成扣除 `value`。

核心判斷如下：

```csharp
if (i == s.Length - 1 || dic[s[i + 1]] <= dic[s[i]])
{
    res += value;
}
else
{
    res -= value;
}
```

### 解法二：共用對應表

`RomanToInt2(string s)` 把符號對應表放在類別欄位 `symbolValues` 中。這樣多次呼叫方法時，不需要每次都重新建立同一份字典。

核心判斷寫法略有不同，但語意相同：

```csharp
if (i < n - 1 && value < symbolValues[s[i + 1]])
{
    res -= value;
}
else
{
    res += value;
}
```

這裡先判斷「目前不是最後一個符號，而且目前值小於下一個值」，符合時套用減法；其餘情況全部視為加法。

### 複雜度分析

令 `n` 為羅馬數字字串長度。

- 時間複雜度：`O(n)`，每個字元只會被掃描一次。
- 空間複雜度：`O(1)`，羅馬符號只有固定七種，對應表大小不隨輸入長度成長。
