# LeetCode 287 - Find the Duplicate Number

使用 .NET Console 專案整理 LeetCode 287 `Find the Duplicate Number` 的三種常見解法：排序、Floyd 快慢指標、值域二分搜尋。專案把題解、可執行示例與驗證輸出集中在 `leetcode_287/Program.cs`，方便直接 build、run，並逐一比較每一種解法的思路與限制。

## 題目連結

- [LeetCode 287. Find the Duplicate Number](https://leetcode.com/problems/find-the-duplicate-number/description/)
- [LeetCode CN 287. 寻找重复数](https://leetcode.cn/problems/find-the-duplicate-number/description/)

## 題目說明

給定一個長度為 `n + 1` 的整數陣列 `nums`，其中每個元素都落在 `[1, n]` 範圍內。因為總共有 `n + 1` 個數字，卻只允許出現 `n` 種不同的值，所以根據抽屜原理，一定至少有一個數字重複出現。

題目要求回傳那個重複的數字，而且還額外限制：

- 不可以修改原始陣列 `nums`
- 只能使用常數額外空間

也就是說，這題不只是「找重複值」，而是要在特定限制下找出重複值。

## 限制條件

- `1 <= n <= 10^5`
- `nums.length == n + 1`
- `1 <= nums[i] <= n`
- `nums` 中只有一個整數會重複出現
- 這個重複整數可能出現兩次以上

## 解題概念與出發點

這題最重要的觀察有三個：

1. 數值範圍固定在 `[1, n]`。
2. 陣列長度卻是 `n + 1`。
3. 只有一個值重複，但可能重複超過兩次。

所以我們可以從三個角度思考：

1. 先排序，讓重複值變成相鄰元素，直接掃描找答案。
2. 把陣列解讀成「索引指向下一個索引」的鏈結結構，用 Floyd Cycle Detection 找環入口。
3. 對值域 `[1, n]` 做二分搜尋，利用「小於等於某個門檻的數量是否過多」來縮小答案範圍。

其中：

- `FindDuplicate` 最直觀，但會修改輸入，不符合題目的嚴格限制。
- `FindDuplicate2` 與 `FindDuplicate3` 都保留輸入，且只使用常數額外空間。
- `FindDuplicate2` 是最貼近經典官方思路的版本。

## 解法比較

| 解法 | 核心想法 | 時間複雜度 | 空間複雜度 | 是否修改輸入 | 是否符合題目額外限制 |
| --- | --- | --- | --- | --- | --- |
| `FindDuplicate` | 排序後找相鄰重複值 | `O(n log n)` | 依排序實作而定 | 會 | 否 |
| `FindDuplicate2` | 把陣列映射成環，找環入口 | `O(n)` | `O(1)` | 不會 | 是 |
| `FindDuplicate3` | 對值域做二分，判斷重複值落在哪半邊 | `O(n log n)` | `O(1)` | 不會 | 是 |

## 解法一：排序法 `FindDuplicate`

### 核心想法

這個方法最容易直覺理解。

先將陣列由小到大排序後，重複的數字一定會排在一起。接著只要從左到右掃描，找第一組相鄰且相等的元素，就能得到答案。

### 為什麼有效

排序會把相同的值集中在一起，所以如果某個數字有重複，它一定會出現在某一對相鄰位置上。只要檢查排序後的相鄰元素，就能找到第一組重複值。

### 範例流程

以 `nums = [1, 3, 4, 2, 2]` 為例：

1. 排序後得到 `[1, 2, 2, 3, 4]`
2. 檢查 `1` 和 `2`，不同，繼續
3. 檢查 `2` 和 `2`，相同
4. 第一組相鄰重複值就是 `2`
5. 回傳 `2`

### 優缺點與題目限制相容性

- 優點是好懂、容易除錯，也很適合作為第一版確認思路。
- 代價是需要排序，因此時間複雜度為 `O(n log n)`。
- 這個方法會直接修改輸入陣列，所以不符合題目「不可修改陣列」的要求。

## 解法二：Floyd 快慢指標 `FindDuplicate2`

### 核心想法

這題的經典解法，是把陣列看成一個隱含的鏈結串列。

做法是把：

- 索引 `i` 視為節點
- `nums[i]` 視為從節點 `i` 指向下一個節點的位置

只要照著 `nums[i]` 一直往下走，就像在沿著 `next` 指標移動。

### 為什麼有效

因為每個值都落在 `[1, n]`，所以每一步都會指向合法索引。又因為總共有 `n + 1` 個位置，卻只有 `n` 種可指向的值，最後一定會形成環。那個造成環入口的值，就是重複數字。

Floyd 演算法分兩段：

1. 用快慢指標先找到環上的相遇點
2. 把其中一個指標移回起點，兩者再同步前進，下一次相遇的位置就是環入口

要先特別記住一件事：第一階段找到的是「環上的相遇點」，不一定是答案。真正的答案是第二階段找到的「環入口」。

### 範例流程

以 `nums = [1, 3, 4, 2, 2]` 為例，先把它看成一條路徑：

```text
0 -> 1 -> 3 -> 2 -> 4 -> 2 -> 4 -> 2 -> ...
```

這張路徑圖可以先看出兩件事：

- `2` 和 `4` 形成了一個環
- 環入口是 `2`，而這個入口值正好就是重複數字

#### 第一階段：先找到環上的相遇點

起點時 `slow = 0`、`fast = 0`，之後：

- `slow` 每次走一步，也就是 `slow = nums[slow]`
- `fast` 每次走兩步，也就是 `fast = nums[nums[fast]]`

| 回合 | slow 新位置 | fast 新位置 |
| --- | --- | --- |
| 1 | `nums[0] = 1` | `nums[nums[0]] = nums[1] = 3` |
| 2 | `nums[1] = 3` | `nums[nums[3]] = nums[2] = 4` |
| 3 | `nums[3] = 2` | `nums[nums[4]] = nums[2] = 4` |
| 4 | `nums[2] = 4` | `nums[nums[4]] = nums[2] = 4` |

此時 `slow == fast == 4`，代表兩者在環上相遇。注意，這裡的 `4` 只是相遇點，還不是答案。

#### 第二階段前，先理解三件事

- 為什麼要把 `slow` 重設成 `0`：因為我們要讓一個指標從「起點」重新出發，另一個指標從「環上的相遇點」出發，兩邊才有機會在環入口碰頭。
- 為什麼第二階段兩邊都只走一步：第一階段已經完成「追上彼此」這件事，第二階段的目標改成「對齊到入口」，所以兩者必須用相同速度前進。
- 為什麼這次相遇才是重複數字：從起點到環入口的距離，剛好會和從相遇點繼續走回環入口的距離對上，因此兩邊第一次再相遇的位置就是環入口。

你可以把第二階段想成：一個人從起點重新出發，另一個人留在環內相遇點，兩人都改成一次走一步。因為第一階段留下來的距離差，剛好會在這一段被抵消，所以他們第一次再碰面的位置就是入口。

#### 第二階段：從起點與相遇點同步前進

- 把 `slow` 重設回 `0`
- `fast` 留在相遇點 `4`
- 兩邊都改成一次走一步

| 回合 | slow 原本位置 | fast 原本位置 | slow 新位置 | fast 新位置 |
| --- | --- | --- | --- | --- |
| 起點 | `0` | `4` | - | - |
| 1 | `0` | `4` | `nums[0] = 1` | `nums[4] = 2` |
| 2 | `1` | `2` | `nums[1] = 3` | `nums[2] = 4` |
| 3 | `3` | `4` | `nums[3] = 2` | `nums[4] = 2` |

兩者在值 `2` 相遇，這個位置就是環入口，也就是重複數字 `2`。

### 優缺點與題目限制相容性

- 優點是不修改輸入、額外空間只有 `O(1)`，而且時間複雜度為 `O(n)`。
- 難點是概念跳躍比較大，尤其是第二階段為什麼要重設指標、為什麼同步一步一步走就能找到答案。
- 這個方法完全符合題目要求的兩個限制，因此是三種方法裡最貼近經典標準答案的版本。

## 解法三：值域二分搜尋 `FindDuplicate3`

### 核心想法

這個方法不是對陣列位置做二分，而是對「答案可能落在哪個值域」做二分。

假設目前猜測中間值 `mid`，我們去統計陣列中有多少個數字 `<= mid`。

### 為什麼有效

如果沒有重複，理論上 `[1, mid]` 最多只能容納 `mid` 個不同數字。

所以：

- 如果 `count <= mid`，代表左半值域沒有擠爆，重複值一定在右半邊
- 如果 `count > mid`，代表左半值域的數量過多，重複值一定在左半邊

這就是這個方法可以持續縮小答案範圍的原因。它的核心不是猜「索引」，而是猜「重複值落在哪一段值域」。

### 範例流程

以 `nums = [3, 1, 3, 4, 2]` 為例：

1. 值域為 `[1, 4]`
2. `mid = 2`
3. 計算 `<= 2` 的數字有 `1, 2`，共 `2` 個
4. 因為 `count = 2`，沒有大於 `mid = 2`，代表左半值域沒有超載
5. 所以答案不在 `[1, 2]`，改搜右半值域 `[3, 4]`
6. 新的 `mid = 3`
7. 計算 `<= 3` 的數字有 `3, 1, 3, 2`，共 `4` 個
8. 因為 `count = 4 > mid = 3`，表示左半值域 `[1, 3]` 已經超載
9. 因此重複值落在 `[3, 3]`
10. 最終答案為 `3`

### 優缺點與題目限制相容性

- 優點是不修改輸入、額外空間為 `O(1)`，而且思路直接建立在抽屜原理上。
- 代價是每輪都要掃描整個陣列，因此整體時間複雜度是 `O(n log n)`。
- 這個方法符合題目限制，但在時間效率上不如 Floyd 快慢指標。
