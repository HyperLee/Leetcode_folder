# LeetCode 45. Jump Game II

[![.NET](https://img.shields.io/badge/.NET-10.0-512BD4)](https://dotnet.microsoft.com/)

這個專案使用 C#/.NET 10 實作 LeetCode 45「Jump Game II」。目標是在保證可抵達最後一格的前提下，計算從索引 `0` 到索引 `n - 1` 所需的最少跳躍次數。

## 題目說明

給定一個 0-indexed 整數陣列 `nums`，一開始位於索引 `0`。每個 `nums[i]` 代表從索引 `i` 最多可以向前跳幾格，也就是可以跳到 `i + j`，其中：

- `0 <= j <= nums[i]`
- `i + j < nums.Length`

請回傳抵達最後一個索引所需的最少跳躍次數。題目保證最後一格一定可達。

## 限制條件

根據題目限制：

- `1 <= nums.length <= 10^4`
- `0 <= nums[i] <= 1000`
- 保證可以抵達 `nums[n - 1]`

參考來源：[LeetCode Wiki - 45. Jump Game II](https://leetcode.doocs.org/en/lc/45/)

## 解題概念

這題的核心是貪婪選擇。每次跳躍不需要真的枚舉所有路徑，而是找出「下一步可以讓後續涵蓋範圍最大」的位置。專案保留兩種寫法：

- `Jump`: 反向貪婪，從終點往前回推可抵達目前目標的最左索引。
- `Jump2`: 正向貪婪，從起點往右掃描目前跳躍區間，並更新下一層可抵達的最遠邊界。

## 解法設計

### 解法一：反向貪婪 `Jump`

從最後一個索引開始，尋找最左側且能跳到目前目標 `position` 的索引 `i`。找到後，把 `position` 更新為 `i`，代表最少路徑中的前一步可以站在這裡。重複回推到索引 `0`，更新次數就是最少跳躍次數。

這裡容易混淆的是：`for (int i = 0; i < position; i++)` 的確是由左到右掃描，但「反向」指的是 `position` 這個目標位置從終點一路往前回推，不是指 `for` 迴圈本身要由右往左跑。

變數角色可以這樣理解：

- `position`：目前這一輪想抵達的目標。一開始是最後一格，之後會不斷往左更新。
- `i`：本輪正在檢查的候選起跳點。它從索引 `0` 開始往右找，直到找到第一個可以跳到 `position` 的位置。
- `steps`：每成功找到一個前一步位置，就代表路徑多了一次跳躍。

為什麼 `i` 要從小到大找？因為這個寫法要找的是「最左側」可抵達目前目標的位置。越左的位置代表越靠近起點，回推後剩下要處理的距離越短。找到第一個符合 `i + nums[i] >= position` 的索引後，這一輪就可以停止，將 `position` 往左更新成該索引。

所以整體流程是：

1. 先假設現在的目標是最後一格。
2. 從左到右找第一個可以跳到目標的位置。
3. 把該位置設成新的目標，等於反向推回「上一跳應該站在哪裡」。
4. 重複直到目標回到索引 `0`。

- 時間複雜度：`O(n^2)`
- 空間複雜度：`O(1)`

範例 `nums = [2,3,1,1,4]`：

| 回合 | 目前目標 `position` | `for` 掃描過程 | 找到後更新 | 跳躍次數 |
| --- | --- | --- | --- | --- |
| 1 | index `4` | `i = 0` 時 `0 + nums[0] = 2`，到不了 index `4`；`i = 1` 時 `1 + nums[1] = 4`，可以抵達 | `position = 1` | 1 |
| 2 | index `1` | `i = 0` 時 `0 + nums[0] = 2`，可以抵達 index `1` | `position = 0` | 2 |

最後 `position` 回到 index `0`，代表已經反向推回起點，結果為 `2`。從正向路徑來看，就是 `0 -> 1 -> 4`。

### 解法二：正向貪婪 `Jump2`

把每次跳躍可涵蓋的索引視為一層。掃描目前層時，用 `maxPosition` 記錄下一層能抵達的最遠位置；當索引走到目前層邊界 `end`，代表必須跳一次，並把邊界更新為 `maxPosition`。

- 時間複雜度：`O(n)`
- 空間複雜度：`O(1)`

範例 `nums = [2,3,1,1,4]`：

| 掃描位置 | 當前值 | 下一層最遠位置 | 是否到達邊界 | 跳躍次數 |
| --- | --- | --- | --- | --- |
| index `0` | `2` | `2` | 是，邊界更新為 `2` | 1 |
| index `1` | `3` | `4` | 否 | 1 |
| index `2` | `1` | `4` | 是，邊界更新為 `4` | 2 |

結果為 `2`。
