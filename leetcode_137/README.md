# LeetCode 137 - Single Number II

這個專案是 LeetCode 137「Single Number II / 只出現一次的數字 II」的 C# console app 解法整理。程式包含可直接執行的範例資料，並示範三種解法：雜湊表統計、位元狀態機、逐 bit 計數。

題目連結：

- [LeetCode - Single Number II](https://leetcode.com/problems/single-number-ii/description/)
- [LeetCode CN - 只出現一次的數字 II](https://leetcode.cn/problems/single-number-ii/description/)

## 題目說明

給定整數陣列 `nums`，其中每個元素都出現三次，只有一個元素恰好只出現一次。請找出這個只出現一次的元素並回傳。

題目要求：

- 時間複雜度必須是線性。
- 額外空間複雜度必須是常數。

## 限制條件

- `1 <= nums.length <= 3 * 10^4`
- `-2^31 <= nums[i] <= 2^31 - 1`
- 陣列中只有一個元素出現一次，其餘每個元素都恰好出現三次。

程式會依序執行三種解法，並對每筆範例印出 `PASS` 或 `FAIL`。目前範例涵蓋：

- LeetCode 範例：`[2, 2, 3, 2]`，答案為 `3`。

- 負數案例：`[-2, -2, 1, 1, 4, 1, -2]`，答案為 `4`。
- 32-bit 符號位案例：`[int.MinValue, 7, 7, 7]`，答案為 `int.MinValue`。
- 單一元素案例：`[42]`，答案為 `42`。

## 解題出發點

如果每個重複元素都出現兩次，直接 XOR 全部元素即可抵銷成答案。但本題的重複次數是三次，單純 XOR 會留下錯誤的位元狀態，因此需要讓每個 bit 的出現次數以 3 為週期歸零。

本專案保留三種解法：

1. 雜湊表統計：最直覺，適合理解題意，但使用額外空間。
2. 位元狀態機：用兩個整數 mask 模擬每個 bit 的 modulo 3 計數，是符合題目要求的主要解法。
3. 逐 bit 計數：逐一統計 32 個 bit 的出現次數，概念清楚，也符合常數空間。

## 解法一：雜湊表統計

方法：`SingleNumber`

### 設計說明

這個解法使用 `Dictionary<int, int>` 記錄每個數字出現幾次。掃描陣列時，如果數字已存在於字典中就將計數加一；否則新增計數為一。完成統計後，再掃描字典，找出 value 等於 `1` 的 key。

這是最容易理解的 baseline，優點是直接對應「出現次數」這個題意；缺點是字典大小會隨不同數字數量增加，不符合題目要求的常數額外空間。

複雜度：

- 時間複雜度：`O(n)`
- 額外空間：`O(k)`，`k` 是不同數字的數量

### 範例演示

輸入：`[2, 2, 3, 2]`

統計流程：

| 讀到的數字 | 字典內容 |
| --- | --- |
| `2` | `{ 2: 1 }` |
| `2` | `{ 2: 2 }` |
| `3` | `{ 2: 2, 3: 1 }` |
| `2` | `{ 2: 3, 3: 1 }` |

最後找到計數為 `1` 的數字是 `3`，因此回傳 `3`。

## 解法二：位元狀態機

方法：`SingleNumber2`

### 設計說明

這是本題最精簡的常數空間解法。核心想法是：對每個 bit 位置來說，它可能被看過 `0` 次、`1` 次、`2` 次；一旦第 `3` 次出現，就應該歸零，因為出現三次的數字不應該留下任何貢獻。

程式用兩個整數 mask 表示所有 bit 的狀態：

- `ones`：目前出現次數對 3 取餘後為 `1` 的 bit。
- `twos`：目前出現次數對 3 取餘後為 `2` 的 bit。

先把這裡用到的 bit operation 看成「一次處理 32 個開關」即可。每個整數都有 32 個 bit，程式不是只追蹤某一個 bit，而是用 `ones` 和 `twos` 兩個整數同時追蹤所有 bit 位置。

| 運算 | 名稱 | 直覺意義 | 在這題的用途 |
| --- | --- | --- | --- |
| `^` | XOR | 兩邊不同才是 `1` | 遇到新的 `1` 時切換狀態 |
| `&` | AND | 兩邊都是 `1` 才保留 | 只保留允許留下的 bit |
| `~` | NOT | 把 `0` 和 `1` 反轉 | 搭配 `&` 清掉不該留下的 bit |

每讀入一個 `num`，每個 bit 都會依照下列狀態循環：

```text
00 -> 01 -> 10 -> 00
```

這裡的兩位狀態可以理解成 `twos ones`：

| 出現次數對 3 取餘 | `twos` 的該 bit | `ones` 的該 bit | 狀態意義 |
| ---: | ---: | ---: | --- |
| `0` | `0` | `0` | 尚未出現，或已經出現三次並歸零 |
| `1` | `0` | `1` | 出現一次，答案候選 bit |
| `2` | `1` | `0` | 出現兩次，等待第三次清除 |

也就是：

- 第一次看到某 bit：從 `00` 變成 `01`，進入 `ones`。
- 第二次看到某 bit：從 `01` 變成 `10`，從 `ones` 移到 `twos`。
- 第三次看到某 bit：從 `10` 變成 `00`，從 `twos` 清掉。

程式中的兩行更新式就是在做這個狀態轉換：

```csharp
ones = (ones ^ num) & ~twos;
twos = (twos ^ num) & ~ones;
```

第一行先用 `ones ^ num` 切換這次出現在 `num` 裡的 bit。如果某 bit 原本不在 `ones`，這次出現後會被放進 `ones`；如果某 bit 原本就在 `ones`，這次出現後會先被移出 `ones`。後面的 `& ~twos` 會把已經屬於 `twos` 的 bit 清掉，避免同一個 bit 同時存在於 `ones` 和 `twos`。

第二行再用更新後的 `ones` 來更新 `twos`。`twos ^ num` 同樣先切換這次出現的 bit；最後的 `& ~ones` 會清掉已經回到 `ones` 的 bit，確保每個 bit 只會落在「出現一次」或「出現兩次」其中一種狀態。

當整個陣列處理完，所有出現三次的數字都被清除，只出現一次的數字會留在 `ones`。

複雜度：

- 時間複雜度：`O(n)`
- 額外空間：`O(1)`

### 範例演示

輸入：`[2, 2, 3, 2]`

用二進位看：

- `2 = 10`
- `3 = 11`

狀態流程：

| 讀到的數字 | `ones` | `twos` | 說明 |
| --- | --- | --- | --- |
| 初始 | `00` | `00` | 尚未看過任何 bit |
| `2` | `10` | `00` | 高位 bit 第一次出現，所以留在 `ones` |
| `2` | `00` | `10` | 高位 bit 第二次出現，所以移到 `twos` |
| `3` | `01` | `00` | 低位 bit 第一次出現；高位 bit 第三次出現後歸零 |
| `2` | `11` | `00` | 高位 bit 再次出現一次，和低位 bit 一起留在 `ones` |

最後 `ones = 11`，也就是十進位 `3`。

## 解法三：逐 bit 計數

方法：`SingleNumber3`

### 設計說明

這個解法把整數視為 32 個 bit，逐一統計每個 bit 位置上有多少個 `1`。因為除了答案之外，其他數字都出現三次，所以每個 bit 的總數如果對 `3` 取餘：

- 餘數為 `0`：答案在這個 bit 是 `0`。
- 餘數為 `1`：答案在這個 bit 是 `1`。

依序處理 0 到 31 bit，將餘數為 1 的位置設回答案即可。這個方法也能處理負數，因為 C# 的 `int` 使用 32-bit two's complement 表示法，包含符號位在內都用同一套 bit 還原流程。

複雜度：

- 時間複雜度：`O(32n)`，可視為 `O(n)`
- 額外空間：`O(1)`

### 範例演示

輸入：`[2, 2, 3, 2]`

只觀察低兩個 bit：

| bit 位置 | 所有數字在該 bit 的 1 次數 | 對 3 取餘 | 答案 bit |
| --- | ---: | ---: | ---: |
| bit 0 | `0 + 0 + 1 + 0 = 1` | `1` | `1` |
| bit 1 | `1 + 1 + 1 + 1 = 4` | `1` | `1` |

答案的低兩個 bit 是 `11`，十進位為 `3`。
