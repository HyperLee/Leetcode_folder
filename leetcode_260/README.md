# LeetCode 260 - Single Number III

這個專案是 LeetCode 260「Single Number III（只出現一次的數字 III）」的 C# 解法整理。程式位於 `leetcode_260/Program.cs`，目前提供兩種解法：

- 方法一：使用雜湊表統計次數，寫法直覺，適合理解題意。
- 方法二：使用位元 XOR 與分組抵消，符合題目要求的線性時間與常數額外空間。

## 題目說明

給定一個整數陣列 `nums`，其中恰好有兩個元素只出現一次，其餘所有元素都恰好出現兩次。請找出這兩個只出現一次的元素。

答案可以用任意順序回傳。

題目要求：

- 時間複雜度必須是線性。
- 額外空間複雜度必須是常數。

## 限制條件

常見 LeetCode 條件如下：

- `2 <= nums.length <= 3 * 10^4`
- `-2^31 <= nums[i] <= 2^31 - 1`
- 陣列中恰好有兩個元素只出現一次。
- 陣列中其餘元素都恰好出現兩次。

## 解題概念與出發點

這題的核心是「成對出現的數字可以被消掉，只留下兩個真正需要找出的數字」。

若只要求找出答案，最直覺的方式是記錄每個數字出現幾次，再挑出出現一次的兩個數字。這很容易理解，但需要額外的雜湊表空間，因此不符合題目指定的常數額外空間。

若要符合題目要求，需要利用 XOR 的性質：

- `a ^ a = 0`：相同數字 XOR 後會互相抵消。
- `a ^ 0 = a`：任何數字和 0 XOR 仍是自己。
- XOR 具交換律與結合律，因此陣列順序不影響最後結果。

把整個陣列全部 XOR 起來後，所有出現兩次的數字都會抵消，最後只會剩下兩個唯一數字的 XOR 結果。接著找出這個結果中任一個為 `1` 的位元，就能知道兩個唯一數字在該位元上不同，進而把陣列分成兩組，讓每組各自留下其中一個唯一數字。

## 方法一：雜湊表統計

### 設計說明

`SingleNumber` 使用 `Dictionary<int, int>` 統計每個整數出現次數。

流程如下：

1. 建立 `counts` 字典。
2. 逐一走訪 `nums`，將每個數字的出現次數加一。
3. 再走訪字典，找出值為 `1` 的 key。
4. 回傳所有只出現一次的數字。

這個方法的優點是直覺、容易除錯，也能自然處理負數、`0`、`int.MinValue` 等整數值。缺點是需要存下不同數字的計數，因此額外空間會隨輸入規模成長。

### 範例演示流程

以 `nums = [1, 2, 1, 3, 2, 5]` 為例：

1. 掃描到 `1`，記錄 `1 -> 1`。
2. 掃描到 `2`，記錄 `2 -> 1`。
3. 再次掃描到 `1`，更新 `1 -> 2`。
4. 掃描到 `3`，記錄 `3 -> 1`。
5. 再次掃描到 `2`，更新 `2 -> 2`。
6. 掃描到 `5`，記錄 `5 -> 1`。
7. 最後字典內容可視為 `{ 1: 2, 2: 2, 3: 1, 5: 1 }`。
8. 取出出現次數為 `1` 的 key，答案為 `[3, 5]`。

### 複雜度

- 時間複雜度：`O(n)`，需要掃描陣列與字典。
- 空間複雜度：`O(n)`，最壞情況下字典會存放多個不同數字。

## 方法二：位元運算 XOR 分組

### 設計說明

`SingleNumber2` 是符合題目要求的解法。

假設兩個只出現一次的數字是 `a` 與 `b`。將整個陣列 XOR 起來：

```text
xorsum = 所有 nums[i] 的 XOR 結果
```

因為所有成對數字都會互相抵消，最後會得到：

```text
xorsum = a ^ b
```

`a` 和 `b` 是不同數字，所以 `a ^ b` 不會是 `0`，也代表它的二進位表示中至少有一個位元是 `1`。任何一個為 `1` 的位元，都表示 `a` 與 `b` 在該位置不同：一個是 `0`，另一個是 `1`。

程式使用最低有效位：

```csharp
int lsb = (xorsum == int.MinValue ? xorsum : xorsum & (-xorsum));
```

一般情況可用 `xorsum & -xorsum` 取出最低位的 `1`。如果 `xorsum` 剛好是 `int.MinValue`，直接取負會溢位，因此保留 `xorsum` 本身作為分組遮罩。

取得 `lsb` 後，再次掃描陣列：

- `(num & lsb) != 0` 的數字放進第一組。
- `(num & lsb) == 0` 的數字放進第二組。

成對出現的數字一定會被分到同一組，因為相同數字的位元完全相同。兩個唯一數字則會被分到不同組。最後每組內各自 XOR，就能分別得到兩個唯一數字。

### 範例演示流程

以 `nums = [1, 2, 1, 3, 2, 5]` 為例：

1. 全部 XOR：

```text
1 ^ 2 ^ 1 ^ 3 ^ 2 ^ 5
= (1 ^ 1) ^ (2 ^ 2) ^ 3 ^ 5
= 0 ^ 0 ^ 3 ^ 5
= 6
```

2. `6` 的二進位可視為 `0110`，最低有效位的 `1` 是 `0010`，所以 `lsb = 2`。

3. 用 `lsb` 分組：

```text
與 lsb 有交集的組：2, 3, 2
與 lsb 沒有交集的組：1, 1, 5
```

4. 各組 XOR：

```text
2 ^ 3 ^ 2 = 3
1 ^ 1 ^ 5 = 5
```

5. 得到答案 `[3, 5]`。由於題目允許任意順序，`[5, 3]` 也正確；專案中的範例輸出會先排序再比對，方便閱讀與驗證。

### `int.MinValue` 範例補充

測試資料包含：

```text
nums = [int.MinValue, 4, 4, 9, 9, 0]
```

成對的 `4` 與 `9` 會抵消，最後剩下：

```text
int.MinValue ^ 0 = int.MinValue
```

在 C# 的 `int` 中，`int.MinValue` 是 `-2147483648`。如果直接做 `-xorsum`，會超出 `int` 可表示範圍，所以程式特別判斷此情況，避免溢位。

### 複雜度

- 時間複雜度：`O(n)`，需要兩次線性掃描。
- 空間複雜度：`O(1)`，只使用固定數量的整數變數。

## 兩種解法比較

| 解法 | 方法 | 時間複雜度 | 空間複雜度 | 是否符合題目常數空間要求 |
| --- | --- | --- | --- | --- |
| 方法一 | 雜湊表統計次數 | `O(n)` | `O(n)` | 否 |
| 方法二 | XOR 抵消並依最低有效位分組 | `O(n)` | `O(1)` | 是 |
