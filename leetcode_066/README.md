## 題目說明

給定一個整數陣列 `digits`，用來表示一個非負大整數。陣列中的每個元素都是一個十進位數字，並且由左到右依序代表最高位到最低位。

任務是將這個大整數加一，並回傳加一後的數字陣列。

範例：

```text
Input:  digits = [1, 2, 3]
Output: [1, 2, 4]
```

`[1, 2, 3]` 代表整數 `123`，加一後是 `124`，因此輸出 `[1, 2, 4]`。

## 限制條件

- `1 <= digits.Length <= 100`
- `0 <= digits[i] <= 9`
- `digits` 不包含前導 `0`

因為長度最多可到 100 位數，不應該把整個陣列轉成 `int`、`long` 或其他固定長度整數型別再加一；那會有溢位風險。比較穩定的做法是直接在陣列上模擬十進位加法。

## 解題概念與出發點

十進位整數加一時，真正需要處理的只有從最低位開始的進位鏈：

- 如果最低位不是 `9`，只要最低位加一即可。
- 如果最低位是 `9`，該位會變成 `0`，並往左邊進位。
- 如果尾端有連續多個 `9`，這些 `9` 都會因進位變成 `0`。
- 如果所有位數都是 `9`，例如 `[9, 9, 9]`，結果會多出一位，變成 `[1, 0, 0, 0]`。

兩種解法都從右往左掃描，時間複雜度都是 `O(n)`。除了全部為 `9` 時需要建立新陣列外，其他情況都直接在輸入陣列上修改。

## 解法一：找出最長的後綴 9

### 設計說明

`PlusOne` 的核心想法是先找出「從尾端開始，第一個不是 `9` 的數字」。

從右往左掃描時，如果目前位置不是 `9`，代表進位會在這個位置停止。此時：

1. 將目前位置加一。
2. 目前位置右側的所有數字原本都必定是 `9`。
3. 這些右側的 `9` 因為進位後都會變成 `0`。
4. 回傳處理完成的陣列。

如果整個掃描過程都找不到非 `9` 的數字，就表示輸入像 `[9]`、`[9, 9]` 或 `[9, 9, 9]`。這時候原陣列長度不夠，需要建立長度加一的新陣列，將第一個元素設為 `1`，其餘位置維持陣列預設值 `0`。

### 範例演示流程

#### 範例 A：沒有進位

```text
Input: [1, 2, 3]
```

流程：

1. 從最後一位 `3` 開始檢查。
2. `3` 不是 `9`，直接加一成 `4`。
3. 右側沒有其他位數需要清零。
4. 回傳 `[1, 2, 4]`。

#### 範例 B：尾端連續 9

```text
Input: [1, 2, 9, 9]
```

流程：

1. 從最後一位開始，看到 `9`，繼續往左找。
2. 下一位仍是 `9`，繼續往左找。
3. 找到 `2`，它不是 `9`，將它加一成 `3`。
4. `2` 右側原本都是後綴 `9`，全部清為 `0`。
5. 回傳 `[1, 3, 0, 0]`。

#### 範例 C：全部都是 9

```text
Input: [9, 9, 9]
```

流程：

1. 從右到左檢查，每一位都是 `9`。
2. 沒有任何位置能讓進位停止。
3. 建立長度為 `4` 的新陣列。
4. 將第一位設為 `1`，其餘位數維持 `0`。
5. 回傳 `[1, 0, 0, 0]`。

### 複雜度

- 時間複雜度：`O(n)`，最差情況需要掃過整個陣列。
- 空間複雜度：一般情況為 `O(1)`；全部為 `9` 時需要 `O(n)` 的新陣列作為答案。

## 解法二：模擬直式加法

### 設計說明

`PlusOne2` 更直接模擬小學直式加法。從最低位開始，每次都先把目前位數加一，接著取 `% 10` 保留個位數。

判斷方式如下：

1. 目前位數加一後，如果 `% 10` 的結果不是 `0`，代表沒有產生進位，可以立即回傳。
2. 如果結果是 `0`，代表原本是 `9`，加一後變成 `10`，目前位數留下 `0`，進位繼續往左。
3. 如果整個陣列都處理完仍沒有回傳，代表所有位數原本都是 `9`。
4. 建立長度加一的新陣列，將第一位設為 `1`。

這個解法不需要另外找後綴 `9` 後再清零，因為每個被進位經過的 `9` 都會在當下被 `% 10` 改成 `0`。

### 範例演示流程

#### 範例 A：沒有進位

```text
Input: [4, 3, 2, 1]
```

流程：

1. 從最後一位 `1` 開始。
2. `1 + 1 = 2`。
3. `2 % 10 = 2`，結果不是 `0`，代表不需要繼續進位。
4. 回傳 `[4, 3, 2, 2]`。

#### 範例 B：尾端連續 9

```text
Input: [1, 2, 9, 9]
```

流程：

1. 最後一位 `9 + 1 = 10`，保留個位數 `0`，往左進位。
2. 下一位 `9 + 1 = 10`，保留個位數 `0`，往左進位。
3. 下一位 `2 + 1 = 3`。
4. `3 % 10 = 3`，結果不是 `0`，進位停止。
5. 回傳 `[1, 3, 0, 0]`。

#### 範例 C：全部都是 9

```text
Input: [9]
```

流程：

1. `9 + 1 = 10`，目前位數變成 `0`。
2. 已經沒有更高位可以進位，但加法仍需要補出新的最高位。
3. 建立長度為 `2` 的新陣列。
4. 將第一位設為 `1`。
5. 回傳 `[1, 0]`。

### 複雜度

- 時間複雜度：`O(n)`，最差情況需要從尾端進位到最前面。
- 空間複雜度：一般情況為 `O(1)`；全部為 `9` 時需要 `O(n)` 的新陣列作為答案。

## 實作備註

- `PlusOne` 與 `PlusOne2` 在一般情況下會修改傳入的 `digits` 陣列。
- `Main` 執行範例時會先複製輸入陣列，再分別傳入兩種解法，避免第一個解法的原地修改影響第二個解法。
