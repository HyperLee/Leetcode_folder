# LeetCode 1980 — Find Unique Binary String（找出不同的二進位字串）

> **難度：** Medium　|　**分類：** String, Array, Hash Table, Backtracking

---

## 題目說明

### 英文

Given an array of strings `nums` containing `n` **unique** binary strings, each of length `n`, return a binary string of length `n` that **does not appear** in `nums`. If there are multiple answers, you may return any of them.

### 中文

給定一個包含 `n` 個**唯一**二進位字串的陣列 `nums`，每個字串長度均為 `n`，回傳一個長度為 `n` 的二進位字串，該字串**不出現**在 `nums` 中。若有多個答案，回傳任何一個均可。

**限制條件**

- `1 <= n <= 16`
- `nums.length == n`
- `nums[i].length == n`
- `nums[i]` 僅由 `'0'` 和 `'1'` 組成
- `nums` 中所有字串均不相同

---

## 範例

### 範例一

```
輸入：nums = ["01", "10"]
輸出："11"（或 "00"，任何不在 nums 中的 2 位元字串皆可）
```

### 範例二

```
輸入：nums = ["00", "01"]
輸出："11"（或 "10"）
```

### 範例三

```
輸入：nums = ["111", "011", "001"]
輸出："101"（或任何不在 nums 中的 3 位元字串）
```

---

## 解題概念與出發點

### 核心洞察

長度為 `n` 的二進位字串共有 `2^n` 種可能，而 `nums` 中恰好有 `n` 個字串。當 `n >= 1` 時，`2^n > n`，因此**必定存在**至少一個不在 `nums` 中的字串，答案一定存在。

### 鴿巢原理

`n` 個格子（nums），要裝 `2^n` 個可能值，至少有 `2^n − n` 個值沒有被佔用。我們只需找出其中一個即可。

### 三種切入方向

| 解法 | 核心思路 | 時間複雜度 | 空間複雜度 |
|------|----------|-----------|-----------|
| 方法一 | 康托對角線（條件翻轉） | O(n) | O(n) |
| 方法二 | 暴力枚舉 + HashSet | O(n²) | O(n) |
| 方法三 | 康托對角線（XOR 位元運算） | O(n) | O(n) |

---

## 三種解法詳解

### 解法一：康托對角線（條件翻轉）

**靈感來源：** 數學家康托（Georg Cantor）證明「實數不可數無限」所使用的對角線論證。

**概念：**  
建構一個字串 `res`，使得對每個位置 `i`：

```
res[i] ≠ nums[i][i]
```

這樣一來，`res` 與 `nums[i]` 至少在第 `i` 個字元上不同，因此 `res` 不等於 `nums` 中的任何一個字串。

**做法：** 逐一讀取對角線元素 `nums[i][i]`，將其翻轉後附加至結果字串。

```csharp
public string FindDifferentBinaryString(string[] nums)
{
    string res = "";
    int n = nums.Length;

    for (int i = 0; i < n; i++)
    {
        if (nums[i][i] == '0')
            res += '1';   // '0' 翻轉為 '1'
        else
            res += '0';   // '1' 翻轉為 '0'
    }
    return res;
}
```

---

### 解法二：暴力枚舉（HashSet 整數映射）

**概念：**  
將 `nums` 中所有二進位字串轉為整數存入 `HashSet`，然後從 `0` 往上找第一個不在集合內的整數，再將其轉回補零至 `n` 位的二進位字串。

**為何有效：** 集合中最多 `n` 個元素，因此最多枚舉到 `val = n` 即可找到答案。

```csharp
public string FindDifferentBinaryString2(string[] nums)
{
    int n = nums.Length;
    HashSet<int> set = new HashSet<int>();

    foreach (string num in nums)
        set.Add(Convert.ToInt32(num, 2));   // 二進位 → 整數

    int val = 0;
    while (set.Contains(val))
        val++;                               // 找第一個缺漏值

    string binary = Convert.ToString(val, 2);
    while (binary.Length < n)
        binary = '0' + binary;              // 補零至 n 位

    return binary;
}
```

---

### 解法三：康托對角線（XOR 位元運算）

**概念：** 與解法一相同，但用 XOR 位元運算一行完成翻轉，更為精簡。

`'0' ^ 1 = '1'`（ASCII 48 XOR 1 = 49）  
`'1' ^ 1 = '0'`（ASCII 49 XOR 1 = 48）

```csharp
public string FindDifferentBinaryString3(string[] nums)
{
    int n = nums.Length;
    char[] ans = new char[n];

    for (int i = 0; i < n; i++)
        ans[i] = (char)(nums[i][i] ^ 1);   // XOR 翻轉對角線字元

    return new string(ans);
}
```

## 解題流程示範

以下用 **`nums = ["111", "011", "001"]`（n = 3）** 為例展示三種解法的完整執行過程。

---

### 解法一：條件翻轉

逐步讀取對角線並翻轉：

| 步驟 `i` | `nums[i]` | 對角線 `nums[i][i]` | 翻轉後 | `res` 累積 |
|----------|-----------|---------------------|--------|-----------|
| 0        | `"111"`   | `'1'`               | `'0'`  | `"0"`     |
| 1        | `"011"`   | `'1'`               | `'0'`  | `"00"`    |
| 2        | `"001"`   | `'1'`               | `'0'`  | `"000"`   |

結果 `res = "000"`，確認 `"000"` 不在 `nums` 中 ✓

---

### 解法二：暴力枚舉

**第一步：** 將字串轉為整數

| `nums[i]` | 十進位整數 |
|-----------|-----------|
| `"111"`   | 7         |
| `"011"`   | 3         |
| `"001"`   | 1         |

`set = {7, 3, 1}`

**第二步：** 從 0 往上枚舉

| `val` | 是否在 `set` 中？ |
|-------|-----------------|
| 0     | 否 → 找到答案！  |

**第三步：** `0` 轉為 3 位二進位字串 → `"000"` ✓

---

### 解法三：XOR 位元翻轉

| 步驟 `i` | `nums[i]` | `nums[i][i]`（ASCII） | `^ 1` 結果（ASCII） | `ans[i]` |
|----------|-----------|----------------------|---------------------|---------|
| 0        | `"111"`   | `'1'`（49）           | 48                  | `'0'`   |
| 1        | `"011"`   | `'1'`（49）           | 48                  | `'0'`   |
| 2        | `"001"`   | `'1'`（49）           | 48                  | `'0'`   |

結果 `ans = ['0','0','0']` → `"000"` ✓

---

以 **`nums = ["01", "10"]`（n = 2）** 再示範：

### 解法一

| `i` | `nums[i][i]` | 翻轉 | `res` |
|-----|-------------|------|-------|
| 0   | `'0'` | `'1'` | `"1"` |
| 1   | `'0'` | `'1'` | `"11"` |

結果 `"11"` ✓（不在 `["01","10"]` 中）

### 解法二

`set = {1, 2}`，從 0 枚舉 → `val = 0` 不在 set → `"00"` ✓

### 解法三

| `i` | `nums[i][i]` | `^ 1` |
|-----|-------------|-------|
| 0   | `'0'` | `'1'` |
| 1   | `'0'` | `'1'` |

結果 `"11"` ✓

## 重點整理

> [!NOTE]
> 方法一與方法三本質相同，均為**康托對角線**；方法三用 XOR 更簡潔，方法一用 `if-else` 更易讀。

> [!TIP]
> 三種方法均正確，但**解法一／三（O(n)）優於解法二（O(n²)）**。實際競賽或面試中推薦優先使用解法三。

> [!IMPORTANT]
> 原始的解法一曾有一個 bug：`tmp` 變數被計算後從未附加到 `res`，導致始終回傳空字串。正確做法是直接對 `res` 進行 `+=` 操作。
