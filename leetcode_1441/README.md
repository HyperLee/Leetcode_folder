## 題目說明

給定一個嚴格遞增的整數陣列 `target` 與整數 `n`。資料流會依序提供 `1, 2, 3, ..., n`，我們只能對堆疊執行以下操作：

- `Push`：讀取資料流中的下一個整數，並把它放到堆疊頂端。
- `Pop`：移除堆疊頂端的整數。

目標是讓堆疊由底到頂等於 `target`，並回傳完成目標所需的操作字串。只要堆疊已等於 `target`，就必須立刻停止，不能繼續讀取資料流。

### 限制條件

- `1 <= target.Length <= 100`
- `1 <= n <= 100`
- `1 <= target[i] <= n`
- `target` 中的數字嚴格遞增

### 基本範例

```text
target = [1, 3]
n = 3
```

資料流是 `1, 2, 3`：

1. 讀到 `1`，它是目標值，執行 `Push` 並保留。
2. 讀到 `2`，它不是目標值，但不能略過讀取，所以先執行 `Push`，再執行 `Pop`。
3. 讀到 `3`，它是目標值，執行 `Push` 並保留。
4. 堆疊成為 `[1, 3]`，立即停止。

因此答案是：

```text
[Push, Push, Pop, Push]
```

## 解題概念與出發點

這題最重要的規則是：**不需要的數字不能直接跳過。**

資料流只能依序讀取。每次讀取都對應一次 `Push`，若該數字不在 `target` 中，才緊接著使用 `Pop` 把它移除。因此每個讀到的數字只有兩種處理方式：

| 數字類型 | 操作 | 最後是否留在堆疊 |
|---|---|---|
| 是目前需要的目標值 | `Push` | 是 |
| 不是目前需要的目標值 | `Push`, `Pop` | 否 |

因為 `target` 嚴格遞增，所以不需要搜尋整個陣列，也不需要真的建立一個堆疊。只要知道「目前讀到哪個數字」以及「下一個需要的目標值」即可產生答案。

令 `last = target[target.Length - 1]`。三種解法最多只需要處理到 `last`，因為讀到最後一個目標值時，堆疊一定已完成，後面的數字都不能再讀。

## 解法一：計算相鄰目標值的缺口

對應方法：`BuildArray`

### 設計動機

這個方法不逐一判斷資料流中的每個數字，而是直接觀察相鄰目標值之間缺少多少數字。

假設前一個已保留的目標值是 `prev`，目前目標值是 `number`。兩者之間需要丟棄的數字數量為：

```text
number - prev - 1
```

每個缺少的數字都產生一組 `Push`, `Pop`；目前的目標值則產生一個 `Push`。

第一次迭代把 `prev` 設為 `0`，因此相同公式也能處理第一個目標值之前的缺口。例如第一個目標值為 `2` 時，`2 - 0 - 1 = 1`，代表數字 `1` 必須先 Push 再 Pop。

### 核心不變量

每次完成一個 `number` 後，操作序列所代表的堆疊，恰好等於目前已走訪過的 `target` 前綴；`prev` 則保存最後一個已完成的目標值。

### 程式流程

1. 將 `prev` 初始化為 `0`。
2. 依序走訪 `target` 中的每個 `number`。
3. 重複 `number - prev - 1` 次，加入 `Push` 和 `Pop`。
4. 加入一個 `Push`，保留目前的 `number`。
5. 將 `prev` 更新為 `number`。

### 範例演示：`target = [1, 3]`

| 目前目標 `number` | 前一目標 `prev` | 缺口 `number - prev - 1` | 本輪加入的操作 | 累積操作 |
|---:|---:|---:|---|---|
| 1 | 0 | 0 | `Push` | `[Push]` |
| 3 | 1 | 1 | `Push`, `Pop`, `Push` | `[Push, Push, Pop, Push]` |

第二輪的第一組 `Push`, `Pop` 用來讀取並丟棄數字 `2`，最後一個 `Push` 才是保留數字 `3`。

### 複雜度

- 時間複雜度：`O(target[^1])`。每個讀到的數字至少產生一次操作。
- 額外空間複雜度：`O(target[^1])`，用來保存回傳的操作序列；若不計輸出空間，額外狀態為 `O(1)`。

## 解法二：讀到最後一個目標值

對應方法：`BuildArray2`

### 設計動機

這個方法從資料流角度出發，但不需要一路跑到 `n`。因為 `target` 嚴格遞增，最後一個目標值 `mx` 一定是完成目標前最後需要讀取的數字，所以只枚舉 `1..mx`。

變數 `i` 指向下一個需要的 `target[i]`：

- 讀到 `x` 時先加入 `Push`。
- 若 `x == target[i]`，代表要保留 `x`，將 `i` 往後移。
- 否則 `x` 不是目標值，再加入 `Pop`。

### 核心不變量

每次處理 `x` 前，`i` 指向尚未完成的第一個目標值；處理完成後，堆疊內容等於 `target[0..i)`。

### 程式流程

1. 取得最後一個目標值 `mx`。
2. 將目標指標 `i` 初始化為 `0`。
3. 依序枚舉 `x = 1..mx`。
4. 每次先加入 `Push`。
5. 若 `x` 是 `target[i]`，移動 `i`；否則加入 `Pop`。

### 範例演示：`target = [1, 3]`

| 讀取值 `x` | 處理前的 `target[i]` | 判斷 | 本輪加入的操作 | 處理後的 `i` | 累積操作 |
|---:|---:|---|---|---:|---|
| 1 | 1 | 相等，保留 `1` | `Push` | 1 | `[Push]` |
| 2 | 3 | 不相等，丟棄 `2` | `Push`, `Pop` | 1 | `[Push, Push, Pop]` |
| 3 | 3 | 相等，保留 `3` | `Push` | 2 | `[Push, Push, Pop, Push]` |

當 `x` 走到 `mx = 3` 時，最後一個目標值已完成，因此迴圈結束。參數 `n` 用來描述合法資料流上限；在輸入符合題目限制時，`mx <= n`，所以這個實作不必直接使用 `n` 控制迴圈。

### 複雜度

- 時間複雜度：`O(target[^1])`。
- 額外空間複雜度：`O(target[^1])`，用來保存回傳結果；若不計輸出空間，指標只占 `O(1)`。

## 解法三：完整模擬資料流

對應方法：`BuildArray3`

### 設計動機

這個方法最貼近題目原文：使用 `i` 表示資料流目前提供的數字，使用 `j` 表示下一個需要的目標位置，同時在迴圈條件中保留資料流上限與完成條件。

```text
i <= n && j < target.Length
```

`i <= n` 表示不能超過資料流；`j < target.Length` 表示目標尚未完成。一旦 `j == target.Length`，就會停止讀取後續數字，符合題目的停止規則。

### 核心不變量

處理每個 `i` 後，堆疊內容等於 `target[0..j)`；`j` 同時等於目前已成功保留的目標值數量。

### 程式流程

1. 令 `m = target.Length`。
2. 同時初始化資料流數字 `i = 1` 與目標指標 `j = 0`。
3. 當 `i <= n` 且 `j < m` 時繼續處理。
4. 先為 `i` 加入 `Push`。
5. 若 `i != target[j]`，加入 `Pop`；否則將 `j` 往後移。
6. 目標完成後，由 `j < m` 的條件立即停止。

### 範例演示：`target = [1, 3]`, `n = 3`

| 資料流 `i` | 目標位置 `j` | 目前目標 `target[j]` | 本輪加入的操作 | 處理後的 `j` | 累積操作 |
|---:|---:|---:|---|---:|---|
| 1 | 0 | 1 | `Push` | 1 | `[Push]` |
| 2 | 1 | 3 | `Push`, `Pop` | 1 | `[Push, Push, Pop]` |
| 3 | 1 | 3 | `Push` | 2 | `[Push, Push, Pop, Push]` |

此時 `j == m == 2`，代表所有目標值都已完成。即使 `n` 更大，迴圈也不會再讀取任何數字。

### 複雜度

- 時間複雜度：`O(target[^1])`。在合法輸入下，完成目標時剛好處理到最後一個目標值。
- 額外空間複雜度：`O(target[^1])`，用來保存回傳結果；若不計輸出空間，額外狀態為 `O(1)`。

## 三種解法比較

| 解法 | 主要視角 | 迴圈依據 | 優點 | 適合理解方式 |
|---|---|---|---|---|
| `BuildArray` | 相鄰目標值的缺口 | 直接走訪 `target` | 不需逐一比較所有資料流數字，公式精簡 | 已理解為何缺口會產生 Push/Pop |
| `BuildArray2` | 資料流與目標指標 | `1..target[^1]` | 流程清楚，直接表達「先 Push，再決定是否 Pop」 | 想從指標配對理解模擬 |
| `BuildArray3` | 題目規則的完整模擬 | `i <= n && j < target.Length` | 停止條件最明確，最貼近題目敘述 | 初次理解題目或希望保留 `n` 的語意 |

三個方法的漸進複雜度相同，也都不需要真的維護堆疊。差異主要是表達方式，而不是效能等級。
