# LeetCode 3129 — Find All Possible Stable Binary Arrays I

**LeetCode Daily Question · 2026-03-09**

[English](https://leetcode.com/problems/find-all-possible-stable-binary-arrays-i/) · [中文](https://leetcode.cn/problems/find-all-possible-stable-binary-arrays-i/)

---

## 題目描述

給定三個正整數 `zero`、`one` 和 `limit`。

若一個二元陣列 `arr` 滿足以下三個條件，則稱為**穩定的（stable）**：

- `arr` 中 `0` 的個數恰好為 `zero`。
- `arr` 中 `1` 的個數恰好為 `one`。
- `arr` 中每個長度大於 `limit` 的子陣列都同時包含 `0` 和 `1`。

回傳穩定二元陣列的總數，結果對 $10^9 + 7$ 取模。

### 限制條件

| 參數 | 範圍 |
|------|------|
| `zero` | `1 ≤ zero ≤ 200` |
| `one` | `1 ≤ one ≤ 200` |
| `limit` | `1 ≤ limit ≤ zero + one` |

---

## 解題概念與出發點

### 等價條件轉換

原始條件：「每個長度 **大於** `limit` 的子陣列同時包含 `0` 和 `1`」

此條件等價於：

> **陣列中不存在長度超過 `limit` 的連續相同元素。**

**證明思路：**  
假設存在一段長度 `limit + 1` 的連續相同元素（例如全為 `0`），則這段子陣列長度 > `limit` 且不包含 `1`，違反條件。反之，若每個長度 > `limit` 的子陣列都含 `0` 和 `1`，則不可能有長度超過 `limit` 的連續相同元素。

這個等價轉換讓我們的動態規劃設計更為清晰：**每次新增一個元素，只需確保語段不違反 `limit` 約束即可。**

---

## 解法：動態規劃

### 狀態定義

| 狀態 | 意義 |
|------|------|
| `dp[i][j][0]` | 已放置 `i` 個 `0`、`j` 個 `1`，且最後一個元素為 `0` 的合法方案數 |
| `dp[i][j][1]` | 已放置 `i` 個 `0`、`j` 個 `1`，且最後一個元素為 `1` 的合法方案數 |

### 基底條件

```
dp[i][0][0] = 1   for i ∈ [0, min(zero, limit)]   // 只有 0，長度未超過 limit
dp[0][j][1] = 1   for j ∈ [0, min(one,  limit)]   // 只有 1，長度未超過 limit
```

其餘預設為 `0`。

### 轉移方程式

**對於 `dp[i][j][0]`（`i ≥ 1`，`j ≥ 1`，在末尾補一個 `0`）：**

$$
dp[i][j][0] = \begin{cases}
dp[i-1][j][0] + dp[i-1][j][1] & \text{if } i \le \text{limit} \\[6pt]
dp[i-1][j][0] + dp[i-1][j][1] - dp[i-\text{limit}-1][j][1] & \text{if } i > \text{limit}
\end{cases}
$$

當 `i > limit` 時，需減去非法方案數 `dp[i-limit-1][j][1]`：

> 這些非法方案的結構是「……最後一個 `1`，緊跟著恰好 `limit` 個 `0`」，再補一個 `0` 就會造成連續 `limit + 1` 個 `0`，違反穩定條件。`dp[i-limit-1][j][1]` 正是「補入那 `limit` 個 `0` 之前，末尾為 `1`」的狀態。

**對於 `dp[i][j][1]`（`j ≥ 1`，`i ≥ 1`，在末尾補一個 `1`）：** 與上方對稱，交換 `i`/`j` 以及 `0`/`1` 的角色：

$$
dp[i][j][1] = \begin{cases}
dp[i][j-1][1] + dp[i][j-1][0] & \text{if } j \le \text{limit} \\[6pt]
dp[i][j-1][1] + dp[i][j-1][0] - dp[i][j-\text{limit}-1][0] & \text{if } j > \text{limit}
\end{cases}
$$

### 最終答案

$$
\text{answer} = \bigl(dp[\textit{zero}][\textit{one}][0] + dp[\textit{zero}][\textit{one}][1]\bigr) \bmod (10^9+7)
$$

## 範例演示

**輸入：** `zero = 2, one = 1, limit = 1`

### Step 1：列舉所有排列

| 陣列 | 是否合法 | 原因 |
|------|----------|------|
| `[0, 0, 1]` | 否 | `[0, 0]` 為長度 2 的連續 `0`，超過 `limit = 1` |
| `[0, 1, 0]` | **是** | 最長連續相同元素長度為 1，符合 `limit = 1` |
| `[1, 0, 0]` | 否 | `[0, 0]` 為長度 2 的連續 `0`，超過 `limit = 1` |

**答案：1**

### Step 2：DP 表格計算過程

**基底初始化（`limit = 1`）：**

```
dp[0][0][0] = 1   // 空陣列前 0 個 0
dp[1][0][0] = 1   // [0]         i ≤ min(2, 1) = 1
dp[0][0][1] = 1   // 空陣列前 0 個 1
dp[0][1][1] = 1   // [1]         j ≤ min(1, 1) = 1
```

**填表 `i = 1, j = 1`（`i = 1 ≤ limit = 1`，`j = 1 ≤ limit = 1`）：**

```
dp[1][1][0] = dp[0][1][0] + dp[0][1][1]
            = 0 + 1 = 1          → 對應方案：[1, 0]

dp[1][1][1] = dp[1][0][1] + dp[1][0][0]
            = 0 + 1 = 1          → 對應方案：[0, 1]
```

**填表 `i = 2, j = 1`（`i = 2 > limit = 1`，`j = 1 ≤ limit = 1`）：**

```
dp[2][1][0] = dp[1][1][0] + dp[1][1][1] - dp[i-limit-1][1][1]
            = dp[1][1][0] + dp[1][1][1] - dp[0][1][1]
            = 1 + 1 - 1 = 1      → 對應方案：[0, 1, 0]
              ^扣除非法方案 [1, 0, 0, ?] → 此處扣去 [1] 後接兩個 0 的情形

dp[2][1][1] = dp[2][0][1] + dp[2][0][0]
            = 0 + 0 = 0          → dp[2][0][0] = 0（2 個連續 0 超過 limit=1）
```

**最終結果：**

```
dp[2][1][0] + dp[2][1][1] = 1 + 0 = 1  ✓
```

## 複雜度分析

| 面向 | 複雜度 |
|------|--------|
| 時間 | $O(\textit{zero} \times \textit{one})$ |
| 空間 | $O(\textit{zero} \times \textit{one})$ |

> [!NOTE]
> 由於每個狀態 `dp[i][j]` 只需要當前行與前一行的資料，理論上可以進一步壓縮至 $O(\textit{one})$ 的空間，但由於題目限制 `zero, one ≤ 200`，原始三維陣列的記憶體佔用相當小，無需額外優化。
