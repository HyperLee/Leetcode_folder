## 題目說明

給定一個 `n x n` 的整數矩陣 `matrix`，請找出任意一條下降路徑的最小總和。

下降路徑可以從第一列的任意元素開始。假設目前位於 `(row, column)`，移動到下一列時只能選擇：

- 左下方：`(row + 1, column - 1)`
- 正下方：`(row + 1, column)`
- 右下方：`(row + 1, column + 1)`

超出矩陣左右邊界的位置不能選擇。到達最後一列後，路徑上所有元素的總和就是該下降路徑的成本。

### 範例

```text
matrix = [[2, 1, 3],
          [6, 5, 4],
          [7, 8, 9]]
```

其中一條最小下降路徑是 `1 -> 5 -> 7`：

```text
1 + 5 + 7 = 13
```

因此答案為 `13`。

## 限制條件

- `1 <= n <= 100`
- `matrix.length == n`
- `matrix[i].length == n`
- `-100 <= matrix[i][j] <= 100`
- 題目保證輸入是非空的正方形矩陣。

在這些限制下，最長路徑只有 `100` 個元素，路徑總和可安全存放在 C# 的 `int` 中。

## 解題概念與出發點

如果直接列舉所有路徑，每走到下一列最多會產生三個選擇，路徑數量會隨矩陣高度呈指數成長。許多不同路徑卻會抵達同一個位置，之後面對完全相同的剩餘問題，因此可以把重複結果保存起來。

對矩陣中的每一格 `(row, column)`，真正需要知道的只有：

> 從第一列走到這一格時，最小的累積總和是多少？

將這個值定義為：

```text
dp[row][column]
```

若不是第一列，能抵達目前位置的合法前驅只可能是左上、正上和右上。狀態轉移因此為：

```text
dp[row][column]
    = matrix[row][column]
    + min(
        dp[row - 1][column - 1],
        dp[row - 1][column],
        dp[row - 1][column + 1]
      )
```

計算時必須排除超出左右邊界的前驅：

- `column == 0`：沒有左上方。
- `column == n - 1`：沒有右上方。
- `n == 1`：第一列同時也是最後一列，唯一元素就是答案。

第一列沒有前驅，所以初始狀態直接複製原矩陣的第一列。最後一列的每一格都代表一條完整下降路徑，答案就是該列的最小值。

## 解法比較

| 解法 | 公開方法 | 時間複雜度 | 額外空間 | 是否修改輸入 | 適合用途 |
| --- | --- | --- | --- | --- | --- |
| 二維動態規劃 | `MinFallingPathSum` | `O(n^2)` | `O(n^2)` | 否 | 保留每一格狀態，最容易觀察與教學 |
| 一維滾動陣列 | `MinFallingPathSumOptimized` | `O(n^2)` | `O(n)` | 否 | 只需要答案時節省記憶體 |

兩種解法都必須查看矩陣中的每一格，所以時間複雜度相同。差異在於是否保留所有已完成列的 DP 結果。

## 解法一：二維動態規劃

### 設計

建立與輸入矩陣相同大小的 `dp`：

1. 將 `matrix[0]` 複製到 `dp[0]`。
2. 從第二列開始，由上往下逐列計算。
3. 每一格先把正上方當作候選最小值。
4. 若左上方存在，將它加入比較。
5. 若右上方存在，也將它加入比較。
6. 把最小前驅累積和加上當前矩陣值。
7. 回傳 `dp` 最後一列的最小值。

這個版本完全保留每一列的歷史狀態，因此很適合用來理解 DP 表格如何形成，也能進一步擴充成回溯實際路徑的版本。

### 範例演示流程

輸入矩陣：

```text
[2, 1, 3]
[6, 5, 4]
[7, 8, 9]
```

第一列直接作為初始狀態：

```text
dp[0] = [2, 1, 3]
```

計算第二列：

| 位置 | 合法前驅 | 計算 | 結果 |
| --- | --- | --- | --- |
| `(1, 0)` | `2, 1` | `6 + min(2, 1)` | `7` |
| `(1, 1)` | `2, 1, 3` | `5 + min(2, 1, 3)` | `6` |
| `(1, 2)` | `1, 3` | `4 + min(1, 3)` | `5` |

```text
dp[1] = [7, 6, 5]
```

計算第三列：

| 位置 | 合法前驅 | 計算 | 結果 |
| --- | --- | --- | --- |
| `(2, 0)` | `7, 6` | `7 + min(7, 6)` | `13` |
| `(2, 1)` | `7, 6, 5` | `8 + min(7, 6, 5)` | `13` |
| `(2, 2)` | `6, 5` | `9 + min(6, 5)` | `14` |

```text
dp[2] = [13, 13, 14]
```

最後一列的最小值為 `13`，所以回傳 `13`。

### 複雜度

- 時間複雜度：`O(n^2)`，每一格只計算一次。
- 空間複雜度：`O(n^2)`，保存完整的 DP 矩陣。

## 解法二：一維滾動陣列

### 為什麼能節省空間？

計算第 `row` 列時只會讀取第 `row - 1` 列，不會再使用更早的列。因此不需要保存完整的二維表格，只需要：

- `previous`：上一列的最小累積和。
- `current`：正在計算的這一列。

每完成一列就交換兩個陣列的角色：

```csharp
(previous, current) = (current, previous);
```

交換的是陣列參考，不是逐一複製元素。交換後，剛完成的 `current` 成為下一輪要讀取的 `previous`；舊的 `previous` 則作為下一輪可覆寫的緩衝區。

### 範例演示流程

仍使用相同矩陣：

```text
[2, 1, 3]
[6, 5, 4]
[7, 8, 9]
```

初始化：

```text
previous = [2, 1, 3]
current  = [0, 0, 0]
```

完成第二列：

```text
current = [7, 6, 5]
```

交換後：

```text
previous = [7, 6, 5]
current  = [2, 1, 3]  // 舊陣列，下一輪的每一格都會被覆寫
```

完成第三列：

```text
current = [13, 13, 14]
```

再次交換後：

```text
previous = [13, 13, 14]
```

`previous` 的最小值為 `13`，所以回傳 `13`。計算結果與二維 DP 完全相同，但只保留兩個長度為 `n` 的陣列。

### 複雜度

- 時間複雜度：`O(n^2)`，仍然需要處理每一格。
- 空間複雜度：`O(n)`，只保存兩列狀態。
