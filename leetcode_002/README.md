# LeetCode 2: Add Two Numbers

以 C# 實作 LeetCode 第 2 題，示範如何用 linked list 完成兩個非負整數的逐位相加。本專案保留兩種解法，方便對照「重用 sum」與「獨立 carry」兩種思考方式。

## 題目說明

給定兩條非空 linked list，每個節點只儲存一位數字，整個數字則以反向順序儲存。也就是說：

- [2 -> 4 -> 3] 代表 342
- [5 -> 6 -> 4] 代表 465

任務是把兩個數字相加，並同樣以反向 linked list 回傳結果，因此答案 [7 -> 0 -> 8] 代表 807。

### 需要處理的重點

- 兩條 linked list 的長度可能不同。
- 某一位相加後可能產生進位。
- 全部節點處理完之後，最高位仍可能需要補上一個 1。
- 節點值都在 0 到 9 之間，且除了數字 0 本身之外，不會有前導零。

## 解題概念與出發點

直觀做法是先把 linked list 轉成整數，相加後再轉回 linked list，但這樣會有兩個問題：

1. 題目故意用 linked list 表示數字，就是要避免直接依賴整數型別的上限。
2. 就算可以轉型，也多了不必要的轉換成本。

更好的方式是直接在 linked list 上模擬直式加法。因為題目已經把個位數放在最前面，所以我們從 head 開始走訪，剛好就是由個位、十位、百位一路往上加，完全不需要先反轉 linked list。

> [!IMPORTANT]
> 這題的核心不是 linked list 技巧本身，而是如何穩定處理進位。
> 只要能把每一輪拆成「取值、相加、留下個位數、保存進位」四個步驟，整題就會非常規律。

每一輪都遵循同樣流程：

1. 取出 l1 與 l2 目前節點的值，如果某一邊已經走完，就把該位視為 0。
2. 把兩個位數與上一輪的進位相加。
3. 將個位數 sum % 10 建立成新節點，接到答案尾端。
4. 將進位更新為 0 或 1，繼續處理下一位。

為了讓結果 linked list 的建立更簡單，兩種解法都使用 dummy head。這樣第一個答案節點不需要特別判斷，最後直接回傳 dummyHead.next 即可。

## 兩種解法總覽

| 方法 | 核心概念 | 優點 | 需要留意的地方 | 時間複雜度 | 空間複雜度 |
| --- | --- | --- | --- | --- | --- |
| AddTwoNumbers | 用同一個 sum 變數同時表示上一輪進位與本輪總和 | 變數較少，寫法精簡 | 讀程式時要理解 sum 在不同時機的角色不同 | O(n) | O(n) |
| AddTwoNumbers2 | 把 carry 抽成獨立變數 | 語意清楚，最接近直式加法 | 變數稍多，但更容易維護 | O(n) | O(n) |

### 解法一：AddTwoNumbers

這個版本把進位資訊放在 sum 裡。進入下一輪時，先看上一輪 sum 是否大於等於 10，如果是，就把 sum 轉成 1；否則轉成 0。接著再把目前 l1 與 l2 的節點值加進來。

這樣做的好處是：

- 不需要另外宣告 carry 變數。
- 整體程式碼較短。
- 仍然可以正確處理不同長度與最後一位進位。

但閱讀時要注意：sum 在迴圈剛開始時代表「上一輪進位」，在加完當前兩個節點後才代表「本輪總和」。這種雙重角色是此版本最需要理解的地方。

適合想練習：

- 精簡狀態管理
- 用較少變數完成 linked list 題目
- 理解進位如何被壓縮在同一個變數中

### 解法二：AddTwoNumbers2

這個版本把 carry 獨立出來，讓每一輪的責任非常固定：

1. 讀取兩個當前位數。
2. 加上 carry。
3. 建立個位數節點。
4. 更新 carry。
5. 指標往後走。

這個版本的優點是：

- 變數意圖清楚。
- 每一輪流程直觀。
- 對除錯、教學與後續維護都更友善。

如果把這題當成 linked list 與直式加法的教學範例，這個版本通常是更推薦的主解，因為思路最容易直接對應到手算流程。

## 流程示範

以下都用同一組輸入示範：

- l1 = [2 -> 4 -> 3]，代表 342
- l2 = [5 -> 6 -> 4]，代表 465
- 正確答案 = [7 -> 0 -> 8]，代表 807

### 解法一流程示範

此方法的關鍵是 sum 先承接上一輪進位，再累加當前位數。

| 輪次 | 進入本輪前的 sum | 轉成進位後 | l1 位數 | l2 位數 | 本輪總和 | 寫入節點 | 目前結果 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 0 | 0 | 2 | 5 | 7 | 7 | [7] |
| 2 | 7 | 0 | 4 | 6 | 10 | 0 | [7 -> 0] |
| 3 | 10 | 1 | 3 | 4 | 8 | 8 | [7 -> 0 -> 8] |

說明：

- 第 2 輪結束後 sum = 10，因此第 3 輪開始時會先被轉成 1，代表上一輪的進位。
- 所有節點處理完後，若 sum 仍大於等於 10，就代表最高位還要再補一個 1。

### 解法二流程示範

此方法把 carry 獨立出來，步驟更像手算。

| 輪次 | carry | l1 位數 | l2 位數 | 本輪總和 | 新 carry | 寫入節點 | 目前結果 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 0 | 2 | 5 | 7 | 0 | 7 | [7] |
| 2 | 0 | 4 | 6 | 10 | 1 | 0 | [7 -> 0] |
| 3 | 1 | 3 | 4 | 8 | 0 | 8 | [7 -> 0 -> 8] |

說明：

- carry 只負責保存下一輪要加上的進位。
- sum 只代表本輪總和，因此閱讀時不需要在同一個變數上切換思維。

> [!TIP]
> 想觀察連續進位時，可以直接執行主程式中的範例 3，輸入會一路把進位往後傳遞到新的最高位。

## 主程式內建測試資料

目前 Main 會執行以下案例：

- 官方常見範例：342 + 465 = 807
- 雙零案例：0 + 0 = 0
- 長度不同且連續進位：9999999 + 9999 = 10009998
- 單邊為 0：81 + 0 = 81
- 單一節點進位：9 + 1 = 10
- 含 0 且最後仍需進位：505 + 595 = 1100

這些案例分別覆蓋：

- 一般加總
- 全零輸入
- linked list 長度不同
- 單輪進位
- 連續進位
- 中間包含 0 的位數組合

## 為什麼保留兩種解法

這個專案的重點不只是解出題目，而是把同一題的兩種思考方式並排展示：

- AddTwoNumbers 適合用來理解如何用較少狀態完成進位控制。
- AddTwoNumbers2 適合用來理解更清楚、可維護性更高的直式加法寫法。

如果你是第一次做這題，建議先理解 AddTwoNumbers2，再回頭看 AddTwoNumbers，會更容易看出兩者其實在做同一件事，只是狀態管理方式不同。
