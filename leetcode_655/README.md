# LeetCode 655. Print Binary Tree

## 題目說明

給定一棵二元樹的根節點 `root`，要建立一個 `m x n` 的字串矩陣 `res`，把整棵樹按照題目指定的位置規則擺進矩陣中。

題目的格式規則如下：

1. 樹的高度為 `height`，而且高度從 `0` 開始計算。
2. 矩陣列數 `m = height + 1`。
3. 矩陣欄數 `n = 2^(height + 1) - 1`。
4. 根節點要放在第一列正中央。
5. 每個節點的左子樹要放在左半邊空間，右子樹要放在右半邊空間。
6. 沒有節點的位置要填入空字串 `""`。

最後回傳這個格式化後的二維字串矩陣。

題目連結：

- LeetCode: [https://leetcode.com/problems/print-binary-tree/description/](https://leetcode.com/problems/print-binary-tree/description/)
- LeetCode CN: [https://leetcode.cn/problems/print-binary-tree/description/](https://leetcode.cn/problems/print-binary-tree/description/)

## 解題概念與出發點

這題的重點不是一般的樹走訪本身，而是要把「樹的結構」正確映射到「矩陣位置」。

先觀察題目規則，可以發現：

1. 矩陣大小完全由樹高決定，所以第一步一定要先知道整棵樹有多高。
2. 每個節點都會落在「目前可用區間的中間位置」。
3. 一旦某個節點被放到中間，剩下的空間自然會被切成左半部和右半部。
4. 左子樹只會出現在左半部，右子樹只會出現在右半部。

因此這題可以有兩個很自然的方向：

1. 使用 DFS，直接以遞迴方式不斷切分左右區間。
2. 使用 BFS，先逐層求出高度，再逐層把節點放到對應座標。

兩種方法本質上都遵守相同的版面規則，只是走訪方式不同。

## 解法一：DFS

### Step 1：先求樹高

本題明確說明高度從 `0` 開始，因此：

- 空節點高度可視為 `-1`
- 葉節點高度為 `0`

遞迴公式為：

```text
height(node) = max(height(node.left), height(node.right)) + 1
```

這樣算出來的高度就能直接套用題目公式：

```text
rows = height + 1
cols = 2^(height + 1) - 1
```

### Step 2：依高度建立空白矩陣

當我們知道 `rows` 和 `cols` 後，就可以先建立一個全部填滿 `""` 的矩陣。

例如樹高為 `2`：

```text
rows = 3
cols = 7
```

那麼矩陣大小就是 `3 x 7`。

### Step 3：DFS 將節點放入矩陣

接著遞迴處理每個節點，並且維護目前可使用的欄位區間 `[left, right]`。

對於任何一個節點：

1. 先找出中間位置 `mid = left + (right - left) / 2`
2. 把當前節點值放到 `matrix[row][mid]`
3. 左子樹遞迴到左半部區間 `[left, mid - 1]`
4. 右子樹遞迴到右半部區間 `[mid + 1, right]`

這個做法等價於題目中的位移公式，但在實作上更容易理解，因為它是直接從「切空間」的角度出發。

## 解法二：BFS

廣度優先搜尋也可以完成這題，而且思路同樣清楚。

核心做法分成兩段：

1. 先透過 BFS 逐層走訪整棵樹，得到高度 `height`。
2. 建立 `m = height + 1`、`n = 2^(height + 1) - 1` 的答案矩陣。
3. 再做一次 BFS，把每個節點與它對應的 `(row, col)` 一起放入佇列。
4. 每取出一個節點，就把節點值放到 `res[row][col]`。
5. 若有左子節點與右子節點，就依題目公式計算偏移量後放入下一層。

### Step 1：使用 BFS 計算高度

這個做法會把每一層的節點數量記錄下來，並在每處理完一層後把高度加一。

因此：

- 空樹高度為 `-1`
- 葉節點高度為 `0`

這和 DFS 的高度定義完全一致，只是改成用佇列逐層處理。

### Step 2：建立空白矩陣

當高度求出後，矩陣大小仍然沿用相同公式：

```text
rows = height + 1
cols = 2^(height + 1) - 1
```

接著先建立一個全部填滿 `""` 的矩陣，等候後續 BFS 放值。

### Step 3：BFS 放入節點

第二次 BFS 時，佇列中除了節點本身，還要額外記錄：

1. 目前所在列 `row`
2. 目前所在欄 `col`

一開始根節點會被放在：

```text
row = 0
col = (cols - 1) / 2
```

當取出某個節點後：

1. 把節點值填到 `matrix[row][col]`
2. 若左子節點存在，放到 `(row + 1, col - 2^(height - row - 1))`
3. 若右子節點存在，放到 `(row + 1, col + 2^(height - row - 1))`

這樣就能完全對應題目給定的擺放規則。

## 範例流程演示

以下面這棵樹為例：

```text
    1
   / \
  2   3
   \
    4
```

### 1. 計算樹高

- 節點 `4` 是葉節點，高度為 `0`
- 節點 `2` 的高度為 `1`
- 節點 `3` 是葉節點，高度為 `0`
- 根節點 `1` 的高度為 `2`

因此：

```text
rows = 2 + 1 = 3
cols = 2^(2 + 1) - 1 = 7
```

矩陣大小為 `3 x 7`。

### 2. 放入根節點

第一列可用範圍是 `[0, 6]`，中點是 `3`，所以：

```text
["", "", "", "1", "", "", ""]
```

### 3. 處理左子樹與右子樹

根節點 `1` 放好後：

- 左子樹可用區間是 `[0, 2]`
- 右子樹可用區間是 `[4, 6]`

第二列：

- 節點 `2` 放在 `[0, 2]` 的中間，也就是欄位 `1`
- 節點 `3` 放在 `[4, 6]` 的中間，也就是欄位 `5`

得到：

```text
["", "2", "", "", "", "3", ""]
```

### 4. 繼續遞迴

節點 `2` 的右子節點 `4` 要放入區間 `[2, 2]`，中點就是 `2`。

第三列：

```text
["", "", "4", "", "", "", ""]
```

### 5. 最終結果

```text
[
  ["", "", "", "1", "", "", ""],
  ["", "2", "", "", "", "3", ""],
  ["", "", "4", "", "", "", ""]
]
```

### 用 DFS 角度理解

因為每一層都遵守同一個規則：

1. 目前節點一定放在目前區間正中央。
2. 放完之後，左右子樹各自遞迴處理自己的半邊空間。

所以整棵樹會自然形成題目要求的對稱結構。

### 用 BFS 角度理解

BFS 的關鍵在於：

1. 先知道整棵樹高度，就能先算出完整矩陣大小。
2. 每個節點一旦知道自己的 `(row, col)`，它的子節點位置就能直接算出來。
3. 由於 BFS 逐層推進，所以可以很直觀地把每一層節點依序放進矩陣。

因此 BFS 並不是改變題目的位置規則，而是改用「逐層推進」的方式來完成同一件事。

## 為什麼這個方法可行

因為每一層都遵守同一個規則：

1. 目前節點一定放在目前區間正中央。
2. 放完之後，左右子樹各自遞迴處理自己的半邊空間。

不論你是用 DFS 遞迴切空間，還是用 BFS 逐層推進座標，最後都會得到同一個排版結果。

## 時間與空間複雜度

### DFS 複雜度

#### DFS 時間複雜度

- 計算樹高需要走訪所有節點一次：`O(n)`
- 填入節點需要再走訪所有節點一次：`O(n)`
- 建立矩陣需要配置 `m x n` 個位置

整體可寫成：

```text
O(m x n + n)
```

#### DFS 空間複雜度

- 答案矩陣需要 `O(m x n)`
- 遞迴呼叫堆疊最差為 `O(height)`

因此整體空間複雜度為：

```text
O(m x n + height)
```

### BFS 複雜度

#### BFS 時間複雜度

- 第一次 BFS 計算高度需要走訪所有節點一次：`O(n)`
- 第二次 BFS 放置節點需要再走訪所有節點一次：`O(n)`
- 建立矩陣仍需要配置 `m x n` 個位置

整體同樣可寫成：

```text
O(m x n + n)
```

#### BFS 空間複雜度

- 答案矩陣需要 `O(m x n)`
- BFS 佇列在最寬一層最差可能達到 `O(width)`

因此整體空間複雜度為：

```text
O(m x n + width)
```

## 兩種解法比較

### 相同點

1. 兩者都必須先知道樹高，才能建立正確大小的矩陣。
2. 兩者都遵守相同的節點擺放規則，因此結果完全一致。
3. 兩者的整體時間複雜度都包含矩陣配置成本 `O(m x n)` 與節點走訪成本 `O(n)`。

### 差異點

1. DFS 比較符合「切左右區間」的直覺，實作簡潔。
2. BFS 比較符合「逐層放節點」的思考方式，和題目給出的座標規則對應更直接。
3. DFS 主要額外成本是遞迴堆疊；BFS 主要額外成本是佇列空間。

### 何時選哪一種

1. 如果想讓程式更精簡、邏輯更接近遞迴切分區間，DFS 會比較適合。
2. 如果想強調逐層走訪與座標推進，或想用非遞迴方式完成，BFS 會更自然。
3. 這題兩者都很合理，專案中保留兩種寫法有助於對照理解。
