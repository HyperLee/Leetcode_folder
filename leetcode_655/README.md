# LeetCode 655. Print Binary Tree

## 題目說明

給定一棵二元樹的根節點 `root`，要建立一個 `m x n` 的字串矩陣 `res`，把整棵樹按照題目指定的位置規則擺進矩陣中。

題目的格式規則如下：

1. 樹的高度為 `height`，而且高度從 `0` 開始計算。
2. 矩陣列數 `m = height + 1`。
3. 矩陣欄數 `n = 2^(height + 1) - 1`。
4. 根節點要放在第一列正中央。
5. 每個節點的左子樹要放在左半邊空間，右子樹要放在右半邊空間。
6. 沒有節點的位置要填入空字串 `""`。

最後回傳這個格式化後的二維字串矩陣。

題目連結：

- LeetCode: https://leetcode.com/problems/print-binary-tree/description/
- LeetCode CN: https://leetcode.cn/problems/print-binary-tree/description/

## 解題概念與出發點

這題的重點不是一般的樹走訪本身，而是要把「樹的結構」正確映射到「矩陣位置」。

先觀察題目規則，可以發現：

1. 矩陣大小完全由樹高決定，所以第一步一定要先知道整棵樹有多高。
2. 每個節點都會落在「目前可用區間的中間位置」。
3. 一旦某個節點被放到中間，剩下的空間自然會被切成左半部和右半部。
4. 左子樹只會出現在左半部，右子樹只會出現在右半部。

這代表我們可以使用 DFS：

1. 第一次 DFS 計算樹高。
2. 依照樹高建立答案矩陣。
3. 第二次 DFS 依序將每個節點放入它所在區間的正中央。

這樣的做法非常直觀，因為它完全符合題目描述的排版邏輯。

## 解法詳細說明

### Step 1：先求樹高

本題明確說明高度從 `0` 開始，因此：

- 空節點高度可視為 `-1`
- 葉節點高度為 `0`

遞迴公式為：

```text
height(node) = max(height(node.left), height(node.right)) + 1
```

這樣算出來的高度就能直接套用題目公式：

```text
rows = height + 1
cols = 2^(height + 1) - 1
```

### Step 2：建立空白矩陣

當我們知道 `rows` 和 `cols` 後，就可以先建立一個全部填滿 `""` 的矩陣。

例如樹高為 `2`：

```text
rows = 3
cols = 7
```

那麼矩陣大小就是 `3 x 7`。

### Step 3：DFS 將節點放入矩陣

接著遞迴處理每個節點，並且維護目前可使用的欄位區間 `[left, right]`。

對於任何一個節點：

1. 先找出中間位置 `mid = left + (right - left) / 2`
2. 把當前節點值放到 `matrix[row][mid]`
3. 左子樹遞迴到左半部區間 `[left, mid - 1]`
4. 右子樹遞迴到右半部區間 `[mid + 1, right]`

這個做法等價於題目中的位移公式，但在實作上更容易理解，因為它是直接從「切空間」的角度出發。

## 範例流程演示

以下面這棵樹為例：

```text
    1
   / \
  2   3
   \
    4
```

### 1. 計算樹高

- 節點 `4` 是葉節點，高度為 `0`
- 節點 `2` 的高度為 `1`
- 節點 `3` 是葉節點，高度為 `0`
- 根節點 `1` 的高度為 `2`

因此：

```text
rows = 2 + 1 = 3
cols = 2^(2 + 1) - 1 = 7
```

矩陣大小為 `3 x 7`。

### 2. 放入根節點

第一列可用範圍是 `[0, 6]`，中點是 `3`，所以：

```text
["", "", "", "1", "", "", ""]
```

### 3. 處理左子樹與右子樹

根節點 `1` 放好後：

- 左子樹可用區間是 `[0, 2]`
- 右子樹可用區間是 `[4, 6]`

第二列：

- 節點 `2` 放在 `[0, 2]` 的中間，也就是欄位 `1`
- 節點 `3` 放在 `[4, 6]` 的中間，也就是欄位 `5`

得到：

```text
["", "2", "", "", "", "3", ""]
```

### 4. 繼續遞迴

節點 `2` 的右子節點 `4` 要放入區間 `[2, 2]`，中點就是 `2`。

第三列：

```text
["", "", "4", "", "", "", ""]
```

### 5. 最終結果

```text
[
  ["", "", "", "1", "", "", ""],
  ["", "2", "", "", "", "3", ""],
  ["", "", "4", "", "", "", ""]
]
```

## 為什麼這個方法可行

因為每一層都遵守同一個規則：

1. 目前節點一定放在目前區間正中央。
2. 放完之後，左右子樹各自遞迴處理自己的半邊空間。

所以整棵樹會自然形成題目要求的對稱結構。

這也是 DFS 很適合這題的原因，因為每個子問題都和原問題有完全相同的形態。

## 時間與空間複雜度

### 時間複雜度

- 計算樹高需要走訪所有節點一次：`O(n)`
- 填入節點也需要走訪所有節點一次：`O(n)`
- 建立矩陣需要配置 `m x n` 個位置

整體來看，可寫成：

```text
O(m x n + n_nodes)
```

若只聚焦在節點 DFS 部分，則是 `O(n_nodes)`。

### 空間複雜度

- 答案矩陣需要 `O(m x n)`
- 遞迴呼叫堆疊在最差情況下需要 `O(height)`

因此整體空間複雜度為：

```text
O(m x n + height)
```
