# LeetCode 3660 - Jump Game IX

## 題目說明

給定一個整數陣列 `nums`。

對於任一索引 `i`，可以依照以下規則跳到另一個索引 `j`：

- 若 `j > i`，只有在 `nums[j] < nums[i]` 時，才能往右跳。
- 若 `j < i`，只有在 `nums[j] > nums[i]` 時，才能往左跳。

題目要找的是：

- 對每一個起點 `i`
- 經過任意次合法跳躍後
- 所有可到達位置中的 `nums` 最大值

最後回傳一個陣列 `ans`，其中 `ans[i]` 表示從索引 `i` 出發時，能到達的最大值。

## 題目真正的重點

這題看起來像是一般的 Jump Game，但實際上不適合用「從每個點暴力模擬跳躍」的方式思考。

更好的切入角度是先觀察兩個位置 `i` 與 `j` 的關係。

假設 `i < j`：

- 若 `nums[i] > nums[j]`，那麼：
  - `i` 可以往右跳到 `j`
  - `j` 也可以往左跳回 `i`
- 也就是說，這兩個點其實是雙向可達的

這代表只要一對位置形成逆序對，也就是：

```text
i < j 且 nums[i] > nums[j]
```

那麼它們之間就存在連通關係。

因此這題可以轉化成：

- 把陣列中的索引依照「可否互相到達」分成若干個連續區間
- 同一個區間中的所有索引，答案都會一樣
- 那個答案就是該區間中的最大值

## 解題出發點

### 1. 不要真的去模擬跳躍

若直接對每個索引做 DFS、BFS 或列出所有可能跳法，時間複雜度很容易來到 `O(n^2)` 甚至更差。

題目雖然在描述跳躍，但本質是在問：

```text
哪些位置屬於同一個可達群組？
```

### 2. 思考什麼時候可以切開成兩段

若我們想在位置 `i` 和 `i + 1` 之間切一刀，必須保證：

- 左邊任何元素都不會和右邊任何元素形成逆序對

也就是：

```text
max(nums[0..i]) <= min(nums[i+1..n-1])
```

只要這個條件成立，就代表：

- 左邊沒有任何值大於右邊某個值
- 沒有跨界逆序對
- 左右兩段彼此不連通

這樣就能安全切開。

## 解法核心：Prefix Max + Suffix Min

### Step 1：預處理右側最小值 `suffixMin`

建立一個陣列 `suffixMin`，其中：

- `suffixMin[i]` 表示從 `i` 到陣列尾端的最小值

例如：

```text
nums      = [4, 2, 5, 1, 3]
suffixMin = [1, 1, 1, 1, 3]
```

這樣當我們掃到某個位置 `i` 時，就能立刻知道右半邊的最小值是多少。

### Step 2：由左到右維護目前區間最大值 `currentBlockMax`

掃描陣列時，持續更新目前區間的最大值：

```text
currentBlockMax = max(目前區間內所有值)
```

### Step 3：判斷是否可以封口

當掃描到索引 `i` 時：

- 若 `i == n - 1`，代表已走到最後一格，這一段一定可以封口
- 或者 `currentBlockMax <= suffixMin[i + 1]`

一旦條件成立，就表示：

- 左側目前這一段，和右側剩餘元素之間不會有跨界逆序對
- 所以目前這段就是一個完整的可達區間

### Step 4：整段填入區間最大值

同一區間中的所有索引彼此可達，因此它們的答案都相同，直接把整段填成 `currentBlockMax` 即可。

## 為什麼這樣做是對的

這個方法建立在一個重要事實上：

- 題目中的合法跳躍，等價於逆序對所形成的雙向連通關係

若某個切點滿足：

```text
左側最大值 <= 右側最小值
```

那就表示不可能存在：

```text
左側某值 > 右側某值
```

也就不可能存在任何跨越這個切點的逆序對。

沒有跨界逆序對，就沒有跨界連通性，所以左右兩邊一定可以分開。

反過來說，只要不滿足這個條件，就表示至少存在一組跨界逆序對，左右兩側仍然連在一起，不能切。

## 演算法流程

1. 若 `nums` 為空陣列，直接回傳空陣列。
2. 建立 `suffixMin`，從右往左填值。
3. 由左到右掃描陣列，維護：
   - `blockStart`：目前區間起點
   - `currentBlockMax`：目前區間最大值
4. 當遇到可切點時：
   - 把 `answer[blockStart..i]` 全部填成 `currentBlockMax`
   - 更新下一段的起點
5. 掃描完成後回傳答案。

## 範例演示

### 範例一

```text
nums = [4, 2, 5, 1, 3]
```

先算 `suffixMin`：

```text
suffixMin = [1, 1, 1, 1, 3]
```

開始由左到右掃描：

### i = 0

```text
目前區間 = [4]
currentBlockMax = 4
右側最小值 = suffixMin[1] = 1
4 <= 1 ? 否
```

不能切。

### i = 1

```text
目前區間 = [4, 2]
currentBlockMax = 4
右側最小值 = suffixMin[2] = 1
4 <= 1 ? 否
```

不能切。

### i = 2

```text
目前區間 = [4, 2, 5]
currentBlockMax = 5
右側最小值 = suffixMin[3] = 1
5 <= 1 ? 否
```

不能切。

### i = 3

```text
目前區間 = [4, 2, 5, 1]
currentBlockMax = 5
右側最小值 = suffixMin[4] = 3
5 <= 3 ? 否
```

仍然不能切。

### i = 4

已到最後一格，這段封口。

整段答案填入 `5`：

```text
ans = [5, 5, 5, 5, 5]
```

### 範例二

```text
nums = [2, 1, 4, 3, 5]
```

先算 `suffixMin`：

```text
suffixMin = [1, 1, 3, 3, 5]
```

掃描過程：

### i = 0

```text
currentBlockMax = 2
suffixMin[1] = 1
2 <= 1 ? 否
```

### i = 1

```text
currentBlockMax = 2
suffixMin[2] = 3
2 <= 3 ? 是
```

可以切，第一段是 `[2, 1]`，答案都填 `2`。

### i = 2

新區間開始：

```text
目前區間 = [4]
currentBlockMax = 4
suffixMin[3] = 3
4 <= 3 ? 否
```

### i = 3

```text
目前區間 = [4, 3]
currentBlockMax = 4
suffixMin[4] = 5
4 <= 5 ? 是
```

可以切，第二段是 `[4, 3]`，答案都填 `4`。

### i = 4

最後一段只有 `[5]`，答案填 `5`。

最後得到：

```text
ans = [2, 2, 4, 4, 5]
```

## 複雜度分析

- 時間複雜度：`O(n)`
  - 一次建立 `suffixMin`
  - 一次掃描並填答案
- 空間複雜度：`O(n)`
  - 需要 `suffixMin`
  - 需要答案陣列

## 實作重點整理

- 這題不是模擬跳躍，而是找可達區間
- 可切條件是：

```text
prefixMax <= suffixMin
```

- 同一區間內所有索引答案都相同
- 每一段答案就是該段最大值
