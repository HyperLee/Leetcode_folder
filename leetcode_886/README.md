# LeetCode 886 — Possible Bipartition（可能的二分法）

**難度**：Medium　｜　**標籤**：圖論、深度優先搜尋（DFS）、二分圖判定

> [LeetCode 題目（英文）](https://leetcode.com/problems/possible-bipartition/description/) ／ [力扣（中文）](https://leetcode.cn/problems/possible-bipartition/description/)

---

## 題目說明

給定 `n` 個人（編號 `1` 到 `n`）與一個「不喜歡」清單 `dislikes`，其中 `dislikes[i] = [a, b]` 表示編號 `a` 的人不喜歡編號 `b` 的人。

**目標**：判斷是否能將所有人分成**兩組**，使得每對「互相不喜歡」的人不在同一組中。若可以，回傳 `true`；否則回傳 `false`。

**限制條件**：

- `1 ≤ n ≤ 2000`
- `0 ≤ dislikes.length ≤ 10000`
- `dislikes[i].length == 2`
- `1 ≤ ai < bi ≤ n`
- 所有 `dislikes[i]` 皆不重複

---

## 解題核心概念

### 問題轉化：二分圖判定

將每個人視為圖中的一個**節點**，若 `a` 不喜歡 `b`，則在 `a` 與 `b` 之間建立一條**無向邊**。

問題因此轉化為：**此無向圖是否為二分圖（Bipartite Graph）？**

> **二分圖**：一個圖可以將所有節點分成兩個獨立集合，使得每條邊的兩端節點分屬不同集合。

**關鍵定理**：一個圖是二分圖，若且唯若圖中不存在**奇數長度的環**。

---

## 解法說明：染色法（DFS）

使用**染色法**模擬分組過程：

- 顏色 `0`：尚未分組
- 顏色 `1`：第一組（紅色）
- 顏色 `2`：第二組（藍色）

### 演算法步驟

1. **建立鄰接表** `g`：對每條 `[a, b]` 邊，同時在 `g[a]` 加入 `b`，並在 `g[b]` 加入 `a`（無向圖）。
2. **依序走訪每個節點**：若節點尚未染色，從此節點啟動 DFS，初始染為顏色 `1`。
3. **DFS 遞迴邏輯**：
   - 將當前節點染色為 `nowcolor`。
   - 對所有相鄰節點（仇人）嘗試染**反色**（`3 XOR nowcolor`）：
     - 若相鄰節點已被染色且與當前節點**同色** → **衝突**，回傳 `false`。
     - 若相鄰節點尚未染色 → 遞迴 DFS 染反色；若遞迴失敗也回傳 `false`。
4. 所有節點皆無衝突，回傳 `true`。

### 反色技巧

使用位元 XOR 運算在顏色 `1` 和 `2` 之間切換，無需 if-else：

```
3 (二進位 11) XOR 1 (二進位 01) = 2 (二進位 10)
3 (二進位 11) XOR 2 (二進位 10) = 1 (二進位 01)
```

### 複雜度分析

| 項目       | 複雜度     |
|------------|------------|
| 時間複雜度 | O(n + m)   |
| 空間複雜度 | O(n + m)   |

其中 `n` 為人數，`m` 為 `dislikes` 的長度（邊數）。

## 流程演示

### 範例一：可以二分（回傳 `true`）

**輸入**：`n = 4`，`dislikes = [[1,2],[1,3],[2,4]]`

**建立鄰接表**：

```
1 → [2, 3]
2 → [1, 4]
3 → [1]
4 → [2]
```

**染色過程**：

```
步驟 1：節點 1 尚未染色，啟動 DFS，將節點 1 染為顏色 1（紅）
步驟 2：節點 1 的仇人是 2、3，嘗試將他們染為顏色 2（藍）
步驟 3：節點 2 的仇人是 1（顏色 1，與 2 不同 ✓）、4（未染色，染為顏色 1（紅））
步驟 4：節點 4 無未處理的仇人，DFS 完成
步驟 5：節點 3 的仇人是 1（顏色 1，與 3 的顏色 2 不同 ✓），DFS 完成
```

**分組結果**：

| 第一組（顏色 1） | 第二組（顏色 2） |
|-----------------|-----------------|
| 1, 4            | 2, 3            |

**結論**：無衝突，回傳 `true` ✓

---

### 範例二：無法二分（回傳 `false`）

**輸入**：`n = 3`，`dislikes = [[1,2],[1,3],[2,3]]`

**建立鄰接表**：

```
1 → [2, 3]
2 → [1, 3]
3 → [1, 2]
```

**染色過程**：

```
步驟 1：節點 1 染為顏色 1（紅）
步驟 2：節點 2 染為顏色 2（藍）
步驟 3：節點 3（是節點 1 的仇人）嘗試染為顏色 2（藍）
步驟 4：檢查節點 3 的仇人 → 節點 2 已是顏色 2（藍），與節點 3 顏色 2 相同 → 衝突！
```

**結論**：1-2-3-1 形成長度為 3 的奇數環，無法二分，回傳 `false` ✗

---

### 範例三：奇數環（回傳 `false`）

**輸入**：`n = 5`，`dislikes = [[1,2],[2,3],[3,4],[4,5],[1,5]]`

圖形結構為 1→2→3→4→5→1，形成長度為 **5** 的奇數環，無法二分。

**結論**：回傳 `false` ✗
