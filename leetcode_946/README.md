# 946. Validate Stack Sequences — 驗證堆疊序列

> LeetCode 第 946 題 — 使用堆疊模擬來驗證 push/pop 序列是否合法。

| 難度 | 標籤 | 語言 |
|------|------|------|
| Medium | Stack, Array, Simulation | C# (.NET 10) |

- [英文題目連結](https://leetcode.com/problems/validate-stack-sequences/description/)
- [中文題目連結](https://leetcode.cn/problems/validate-stack-sequences/description/)

## 題目描述

給定兩個整數陣列 `pushed` 和 `popped`，其中每個陣列的元素**各不相同**。

判斷此序列是否可能是對一個**初始為空的堆疊**依序執行 push 與 pop 操作的結果。若合法則回傳 `true`，否則回傳 `false`。

### 範例

```
輸入: pushed = [1,2,3,4,5], popped = [4,5,3,2,1]
輸出: true

輸入: pushed = [1,2,3,4,5], popped = [4,3,5,1,2]
輸出: false
```

### 限制條件

- `1 <= pushed.length <= 1000`
- `0 <= pushed[i] <= 1000`
- `pushed` 的所有元素**互不相同**
- `popped.length == pushed.length`
- `popped` 是 `pushed` 的一個排列

## 解題概念與思路

### 核心觀察

所有元素一定按照 `pushed` 的順序依次入堆疊，**關鍵在於何時出堆疊**。

由於堆疊中元素互不相同，當堆疊頂端恰好等於 `popped` 中下一個期望彈出的元素時，**必須立刻彈出**——因為之後再 push 新元素會覆蓋堆疊頂端，導致順序永遠無法匹配。

### 解法：堆疊模擬

利用一個輔助堆疊，逐步模擬 push/pop 過程：

1. **依序將 `pushed` 的每個元素推入堆疊。**
2. **每次推入後，持續檢查堆疊頂端是否等於 `popped` 當前指標指向的元素；若相同就彈出，並將指標後移。**
3. **遍歷結束後，若堆疊為空代表序列合法，回傳 `true`；否則回傳 `false`。**

### 複雜度

| | 值 | 說明 |
|---|---|---|
| **時間** | O(n) | 每個元素最多入堆疊與出堆疊各一次 |
| **空間** | O(n) | 輔助堆疊最多存放 n 個元素 |

## 演示流程

以 `pushed = [1,2,3,4,5]`、`popped = [4,5,3,2,1]` 為例，逐步模擬：

| 步驟 | 操作 | 堆疊狀態 | popped 指標 (j) | 說明 |
|:----:|------|:--------:|:------:|------|
| 1 | push 1 | `[1]` | 0 | 堆疊頂 1 ≠ popped[0]=4，不彈出 |
| 2 | push 2 | `[1,2]` | 0 | 堆疊頂 2 ≠ 4，不彈出 |
| 3 | push 3 | `[1,2,3]` | 0 | 堆疊頂 3 ≠ 4，不彈出 |
| 4 | push 4 | `[1,2,3,4]` | 0 | 堆疊頂 4 == popped[0]=4 ✅ |
| 4a | pop 4 | `[1,2,3]` | 1 | 堆疊頂 3 ≠ popped[1]=5，停止彈出 |
| 5 | push 5 | `[1,2,3,5]` | 1 | 堆疊頂 5 == popped[1]=5 ✅ |
| 5a | pop 5 | `[1,2,3]` | 2 | 堆疊頂 3 == popped[2]=3 ✅ |
| 5b | pop 3 | `[1,2]` | 3 | 堆疊頂 2 == popped[3]=2 ✅ |
| 5c | pop 2 | `[1]` | 4 | 堆疊頂 1 == popped[4]=1 ✅ |
| 5d | pop 1 | `[]` | 5 | 堆疊為空，結束 |

最終堆疊為空 → 回傳 **true** ✅

---

再看反例 `pushed = [1,2,3,4,5]`、`popped = [4,3,5,1,2]`：

| 步驟 | 操作 | 堆疊狀態 | j | 說明 |
|:----:|------|:--------:|:-:|------|
| 1–3 | push 1,2,3 | `[1,2,3]` | 0 | 堆疊頂 ≠ 4 |
| 4 | push 4 | `[1,2,3,4]` | 0 | 堆疊頂 4 == popped[0]=4 ✅ |
| 4a | pop 4 | `[1,2,3]` | 1 | 堆疊頂 3 == popped[1]=3 ✅ |
| 4b | pop 3 | `[1,2]` | 2 | 堆疊頂 2 ≠ popped[2]=5，停止 |
| 5 | push 5 | `[1,2,5]` | 2 | 堆疊頂 5 == popped[2]=5 ✅ |
| 5a | pop 5 | `[1,2]` | 3 | 堆疊頂 2 ≠ popped[3]=1，停止 |

遍歷結束，堆疊仍有 `[1,2]` → 回傳 **false** ❌

> [!NOTE]
> 元素 1 被壓在 2 下面，無法在 2 之前彈出，因此 `popped` 中 1 出現在 2 之前是不可能的。
