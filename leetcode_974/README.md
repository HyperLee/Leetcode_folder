## 題目說明

給定一個整數陣列 `nums` 和整數 `k`，計算總和可以被 `k` 整除的非空連續子陣列數量。

子陣列必須是原陣列中一段連續的元素。例如 `[5, 0, -2, -3]` 是 `[4, 5, 0, -2, -3, 1]` 的子陣列，但 `[4, 0, -3]` 不是，因為元素在原陣列中不連續。

- [LeetCode 英文題目](https://leetcode.com/problems/subarray-sums-divisible-by-k/description/)
- [LeetCode 中文題目](https://leetcode.cn/problems/subarray-sums-divisible-by-k/description/)

### 限制條件

- `1 <= nums.Length <= 3 * 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `2 <= k <= 10^4`

在最大陣列長度下，全部子陣列的數量是 `30000 * 30001 / 2 = 450015000`，仍在 C# `int` 可表示的範圍內。

### 官方範例

範例一：

```text
輸入：nums = [4, 5, 0, -2, -3, 1], k = 5
輸出：7
```

符合條件的七個子陣列為：

```text
[4, 5, 0, -2, -3, 1]
[5]
[5, 0]
[5, 0, -2, -3]
[0]
[0, -2, -3]
[-2, -3]
```

範例二：

```text
輸入：nums = [5], k = 9
輸出：0
```

## 解題概念與出發點

### 1. 為什麼先想到前綴和？

暴力解法會列舉所有起點與終點，再計算每個子陣列的總和。即使利用前綴和把每次區間求和降到 `O(1)`，仍有 `O(n^2)` 組區間；當 `nums.Length` 到 `30000` 時，這個成本太高。

定義前綴和：

```text
P[0] = 0
P[i + 1] = nums[0] + nums[1] + ... + nums[i]
```

索引區間 `[i, j)` 的子陣列總和就等於：

```text
P[j] - P[i]
```

### 2. 問題真正需要的是餘數

若子陣列總和能被 `k` 整除：

```text
(P[j] - P[i]) % k = 0
```

這等價於：

```text
P[j] % k = P[i] % k
```

因此不必逐一檢查所有子陣列，只要統計「前綴和除以 `k` 的餘數出現幾次」。每一對相同餘數的前綴和，都對應一個總和可被 `k` 整除的非空連續子陣列。

> [!IMPORTANT]
> 必須先把空前綴 `P[0] = 0` 的餘數 `0` 記錄一次。否則像 `[5]` 搭配 `k = 5` 這種從索引 `0` 開始的合格子陣列不會被計入。

### 3. 為什麼要正規化負餘數？

C# 的 `%` 是餘數運算，結果可能跟被除數一樣是負值。例如：

```text
-1 % 5 = -1
```

但在模 `5` 的同餘分類裡，`-1` 與 `4` 屬於同一組，所以程式使用：

```csharp
if (remainder < 0)
{
    remainder += k;
}
```

將餘數統一到 `[0, k - 1]`。主要範例走到 `-3` 時，前一個餘數是 `2`，`(2 + -3) % 5 = -1`，正規化後便回到餘數 `4`。

## 兩種解法比較

| 程式方法 | 官方題解對應 | 核心做法 | 時間複雜度 | 額外空間 |
| --- | --- | --- | --- | --- |
| `SubarraysDivByK` | 官方解法二 | 每得到一個餘數，立即加入先前相同餘數的次數 | `O(n)` | `O(k)` |
| `SubarraysDivByK2` | 官方解法一 | 先統計每個餘數的總次數，再套組合公式 | `O(n)` | `O(min(n, k))` |

兩種方法運用相同的數學關係，差別只在「何時計算答案」：

- `SubarraysDivByK` 在走訪陣列時即時累加，使用長度為 `k` 的陣列，索引就是餘數。
- `SubarraysDivByK2` 先用 `Dictionary<int, int>` 保存實際出現的餘數，最後才計算每一組能形成幾對。

> [!NOTE]
> 方法名稱中的 `2` 是專案原有的多解法命名慣例；`SubarraysDivByK2` 實際對應官方題解中的「解法一」。

## 解法一：邊遍歷邊統計

對應方法：`SubarraysDivByK(int[] nums, int k)`。

### 設計步驟

1. 建立長度為 `k` 的 `counts`，其中 `counts[r]` 表示先前有多少個前綴和的餘數是 `r`。
2. 設定 `counts[0] = 1`，代表尚未取用任何元素的空前綴。
3. 每讀取一個數字，就更新前綴和餘數並正規化負值。
4. 若目前餘數是 `r`，則先前每一個餘數同為 `r` 的前綴，都能和目前前綴形成一個合格子陣列，因此把 `counts[r]` 加入答案。
5. 最後才將目前前綴記入 `counts[r]`，避免把同一個前綴和自己配對成空子陣列。

### 範例演示流程

輸入：`nums = [4, 5, 0, -2, -3, 1]`、`k = 5`。

初始化：`counts[0] = 1`、`result = 0`。

| `i` | `nums[i]` | 正規化後餘數 | 先前相同餘數次數 | 累計答案 | 更新後該餘數次數 |
| ---: | ---: | ---: | ---: | ---: | ---: |
| 0 | 4 | 4 | 0 | 0 | 1 |
| 1 | 5 | 4 | 1 | 1 | 2 |
| 2 | 0 | 4 | 2 | 3 | 3 |
| 3 | -2 | 2 | 0 | 3 | 1 |
| 4 | -3 | 4 | 3 | 6 | 4 |
| 5 | 1 | 0 | 1 | 7 | 2 |

最終答案為 `7`。

以 `i = 4` 為例，目前餘數是 `4`，先前已經出現三個餘數為 `4` 的前綴和，所以這一步直接新增三個合格子陣列，答案由 `3` 增加到 `6`。

### 複雜度

- 時間複雜度：`O(n)`，每個元素只處理一次。
- 空間複雜度：`O(k)`，`counts` 固定配置 `k` 個位置。

## 解法二：統計完成後套用組合公式

對應方法：`SubarraysDivByK2(int[] nums, int k)`。

### 設計步驟

1. 使用字典統計每一種前綴和餘數的出現次數。
2. 同樣先放入空前綴：餘數 `0` 出現一次。
3. 走訪完整個陣列後，若某個餘數出現 `count` 次，就從這些前綴和中任選兩個。
4. 可形成的配對數量是：

```text
C(count, 2) = count * (count - 1) / 2
```

5. 將所有餘數組的配對數量相加。

### 範例演示流程

輸入：`nums = [4, 5, 0, -2, -3, 1]`、`k = 5`。

包含空前綴的完整前綴和為：

```text
前綴和：0, 4, 9, 9, 7, 4, 5
餘數：  0, 4, 4, 4, 2, 4, 0
```

統計完成後：

| 餘數 | 出現次數 `count` | 組合數 `count * (count - 1) / 2` | 對答案的貢獻 |
| ---: | ---: | ---: | ---: |
| 0 | 2 | `2 * 1 / 2` | 1 |
| 2 | 1 | `1 * 0 / 2` | 0 |
| 4 | 4 | `4 * 3 / 2` | 6 |

總數為 `1 + 0 + 6 = 7`。

餘數 `4` 出現四次，代表可以挑選六對不同位置的前綴和；每一對都界定出一段總和能被 `5` 整除的非空連續子陣列。

### 複雜度

- 時間複雜度：`O(n)`。第一次走訪輸入，第二次只走訪實際出現的餘數種類；種類數不會超過 `n + 1`。
- 空間複雜度：`O(min(n, k))`，字典只保存實際出現的餘數，最多有 `k` 種。

## 如何選擇解法？

- 想要較少配置成本、可直接用餘數當陣列索引：選擇 `SubarraysDivByK`。
- 想先清楚觀察「餘數分組」再套組合公式：`SubarraysDivByK2` 較接近數學推導。
- 在題目限制下兩者都能通過；目前主控台會對相同輸入執行兩種方法並分別驗證。

兩個方法都只讀取 `nums`，不會排序或修改輸入陣列。
