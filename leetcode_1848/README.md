# LeetCode 1848

## Minimum Distance to the Target Element

以 C# 實作 LeetCode 第 1848 題，除了最直觀的線性掃描，也補充了「由 start 向兩側擴散」與「建立索引後二分搜尋」兩種寫法，方便比較不同情境下的取捨。

> [!NOTE]
> 題目保證 target 一定存在於陣列中，因此解法可以專注在「如何找出最小距離」，不必額外處理找不到答案的分支。

[題目說明](https://leetcode.com/problems/minimum-distance-to-the-target-element/) • [專案程式碼](./leetcode_1848/Program.cs)

## 題目在問什麼

給定一個整數陣列 nums、一個目標值 target，還有一個起始索引 start。

你要在陣列中找出某個索引 i，使得：

- nums[i] == target
- abs(i - start) 最小

最後回傳這個最小距離。

換句話說，這題不是要找 target 第一次出現在哪裡，也不是要找 target 的值，而是要找「哪一個 target 所在位置，距離 start 最近」。

## 解題概念與出發點

一開始最重要的觀察是：

只要某個位置不是 target，它就不可能影響答案。

因此，這題真正需要比較的，其實只有每一個 target 出現位置和 start 之間的距離。

如果只處理單次查詢，最自然的做法就是直接掃描所有位置；但若你想利用 start 的位置提早結束，或面對同一個 nums 的多次查詢，也可以改寫成其他策略。

可以把這題拆成三種思路：

1. 線性掃描：逐一檢查整個陣列，保留最小距離。
2. 由 start 向外擴散：按照距離由小到大檢查左右位置，第一個命中就回傳。
3. 建立索引後二分搜尋：先記錄每個值出現的位置，再只對 target 的索引列表找最近位置。

三種方法都圍繞同一個核心：答案本質上就是「所有候選位置中的最小距離」。差別只在於你是一次把候選看完，還是按照距離順序檢查，或先把候選位置整理成方便查詢的結構。

> [!TIP]
> 單次查詢下，線性掃描仍然是最簡潔且最穩定的解法；另外兩種寫法則分別偏向「提早返回」與「支援多次查詢」的情境。

## 解法一：線性掃描

這是單次查詢最直觀，也最穩定的版本。

### 步驟 1：建立最小距離初始值

先用一個變數記錄目前找到的最小距離，初始值設成 int.MaxValue。這代表一開始還沒有找到任何有效答案。

### 步驟 2：從左到右掃描陣列

使用 for 迴圈逐一檢查每個索引。

### 步驟 3：只在遇到 target 時計算距離

如果 nums[index] 不等於 target，就直接跳過。

如果相等，表示這個位置是有效候選，接著計算：

```csharp
Math.Abs(index - start)
```

這就是目前候選位置與 start 的距離。

### 步驟 4：更新最小值

把新算出的距離和目前記錄的最小距離相比，保留較小者。

```csharp
minDistance = Math.Min(minDistance, Math.Abs(index - start));
```

### 步驟 5：回傳答案

整個陣列掃描完後，minDistance 就是所有候選位置中的最小距離，也就是題目答案。

## 為什麼這個解法成立

因為題目要求的是：

$$
\min\left(|i - start|\right) \quad \text{where} \quad nums[i] = target
$$

只要把所有滿足 nums[i] == target 的索引都檢查一遍，再從對應距離中取最小值，就一定能得到正確答案。

線性掃描剛好完整覆蓋所有候選位置，而且每個位置只看一次，因此既正確又簡潔。

### 複雜度

- 時間複雜度：O(n)
- 空間複雜度：O(1)

其中 n 是 nums 的長度。

### 範例流程演示

以這組資料為例：

```text
nums = [1, 2, 3, 4, 5]
target = 5
start = 3
```

掃描流程如下：

1. index = 0，nums[0] = 1，不是 target，跳過。
2. index = 1，nums[1] = 2，不是 target，跳過。
3. index = 2，nums[2] = 3，不是 target，跳過。
4. index = 3，nums[3] = 4，不是 target，跳過。
5. index = 4，nums[4] = 5，符合 target。
6. 計算距離：abs(4 - 3) = 1。
7. 更新最小距離為 1。
8. 掃描結束，回傳 1。

所以答案是：

```text
1
```

再看一個距離為 0 的案例：

```text
nums = [1]
target = 1
start = 0
```

因為 start 本身就是 target 所在位置，距離為：

$$
|0 - 0| = 0
$$

所以答案直接是 0。

## 解法二：由 start 向兩側擴散

這個方法不再先看完整個陣列，而是把 start 當成中心點，按照距離由小到大向外檢查。

### 步驟 1：先檢查距離為 0 的位置

也就是先看 nums[start] 是否就是 target。

如果是，答案立刻就是 0。

### 步驟 2：逐步擴大搜尋半徑

若距離 0 沒找到，就檢查距離為 1 的兩個位置：start - 1 與 start + 1。

接著再檢查距離為 2 的兩個位置，以此類推。

### 步驟 3：第一個找到的 target 直接回傳

因為搜尋順序本來就是從最近的距離往外擴散，所以第一個命中的 target，距離一定是最小值，不需要再繼續掃描其他位置。

## 為什麼這個解法成立

這個方法其實是把所有可能的距離按照順序列出來：

$$
0, 1, 2, 3, \dots
$$

每一輪只檢查與 start 相距剛好為目前距離的位置。

因此，只要在距離 d 時第一次找到 target，就代表不存在比 d 更小的合法距離，答案自然就是 d。

### 複雜度

- 時間複雜度：O(n)
- 空間複雜度：O(1)

雖然最壞情況仍然要看到整個陣列，但如果答案靠近 start，通常可以提早結束。

### 範例流程演示

以這組資料為例：

```text
nums = [5, 1, 2, 3, 5, 4, 5]
target = 5
start = 5
```

檢查流程如下：

1. 距離 = 0，檢查索引 5，nums[5] = 4，不是 target。
2. 距離 = 1，先檢查索引 4，nums[4] = 5，找到 target。
3. 因為這是第一個被找到的距離，所以最小距離就是 1。
4. 直接回傳 1，不需要再繼續看索引 6 或更遠的位置。

所以答案是：

```text
1
```

## 解法三：建立索引後二分搜尋

這個方法會先把每個數值出現的位置整理起來，之後只針對 target 的索引列表做查詢。

### 步驟 1：預先建立索引表

先掃描一次 nums，把每個數值出現的索引存到對應列表中。

例如：

```text
nums = [5, 1, 2, 3, 5, 4, 5]
```

可以建立出：

```text
5 -> [0, 4, 6]
1 -> [1]
2 -> [2]
3 -> [3]
4 -> [5]
```

### 步驟 2：取出 target 的索引列表

若 target = 5，就只需要看：

```text
[0, 4, 6]
```

這份列表天然是遞增的，因為原本就是從左到右掃描陣列建立出來的。

### 步驟 3：對 start 做二分搜尋

使用二分搜尋找出 start 應該插入的位置。

如果剛好命中某個 target 索引，代表 start 本身就是答案，直接回傳 0。

如果沒有剛好命中，只要比較插入點左邊與右邊最近的兩個索引即可。

### 步驟 4：比較左右鄰近候選

因為索引列表是排序好的，所以距離 start 最近的 target，一定只可能是：

- 插入點左邊最近的一個索引
- 插入點右邊最近的一個索引

比較這兩者與 start 的距離後，就能得到最小值。

## 為什麼這個解法成立

設 target 的索引列表為遞增序列：

$$
i_1 < i_2 < i_3 < \dots < i_k
$$

二分搜尋找到的插入點，會把列表切成「小於 start」與「大於 start」兩部分。

在排序序列中，距離 start 最近的元素，不可能跳過鄰居而跑到更遠的位置，因此只需要比較插入點兩側最接近的候選即可。

### 複雜度

- 預處理時間複雜度：O(n)
- 單次查詢時間複雜度：O(log k)
- 空間複雜度：O(n)

其中 n 是 nums 的長度，k 是 target 出現的次數。

> [!NOTE]
> 如果只查一次，預處理成本算進去後，整體仍然接近 O(n)。這個方法真正的價值是在同一個 nums 需要被查很多次時，可以重複使用索引表。

### 範例流程演示

以這組資料為例：

```text
nums = [5, 1, 2, 3, 5, 4, 5]
target = 5
start = 5
```

查詢流程如下：

1. 先建立索引表，得到 target = 5 的索引列表 [0, 4, 6]。
2. 對 start = 5 做二分搜尋，發現它會插在索引值 4 與 6 之間。
3. 比較左側候選：abs(4 - 5) = 1。
4. 比較右側候選：abs(6 - 5) = 1。
5. 取兩者最小值，得到答案 1。

所以答案是：

```text
1
```
