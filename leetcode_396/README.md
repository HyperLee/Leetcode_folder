# 396. Rotate Function

## 題目說明

給定一個長度為 `n` 的整數陣列 `nums`，定義旋轉函數 `F(k)` 如下：

- 將陣列 `nums` 順時針旋轉 `k` 次，得到 `arrk`
- 計算：

```text
F(k) = 0 * arrk[0] + 1 * arrk[1] + 2 * arrk[2] + ... + (n - 1) * arrk[n - 1]
```

我們的目標是求出：

```text
max(F(0), F(1), F(2), ..., F(n - 1))
```

題目保證答案可以放進 32 位元整數。

題目連結：

- LeetCode: <https://leetcode.com/problems/rotate-function/>
- 力扣: <https://leetcode.cn/problems/rotate-function/>

## 解題概念與出發點

最直觀的做法是：

1. 每旋轉一次陣列，就重新計算一次對應的旋轉函數值
2. 總共要算 `n` 次
3. 每次計算又要走訪整個陣列

這樣的時間複雜度會是 `O(n^2)`，當 `n` 很大時效率不夠好。

這題的關鍵在於不要真的去旋轉陣列，而是找出相鄰兩個旋轉函數之間的關係。

設：

- `numSum` = 陣列所有元素總和
- `F(0)` = 原始陣列的旋轉函數值

那麼：

```text
F(0) = 0 * nums[0] + 1 * nums[1] + ... + (n - 1) * nums[n - 1]
F(1) = F(0) + numSum - n * nums[n - 1]
```

更一般地，當 `1 <= k < n` 時：

```text
F(k) = F(k - 1) + numSum - n * nums[n - k]
```

有了這個公式後，我們只需要：

1. 先算出 `numSum`
2. 先算出 `F(0)`
3. 利用遞推公式一路推出 `F(1)`、`F(2)`、...、`F(n - 1)`
4. 過程中持續更新最大值

這樣就能把時間複雜度降到 `O(n)`。

## 解法詳細說明

### 1. 計算陣列總和

先把陣列每個元素加總，得到：

```text
numSum = nums[0] + nums[1] + ... + nums[n - 1]
```

這個總和會在每一次遞推時重複用到。

### 2. 計算初始值 F(0)

原始狀態下不旋轉：

```text
F(0) = 0 * nums[0] + 1 * nums[1] + 2 * nums[2] + ... + (n - 1) * nums[n - 1]
```

這是之後所有遞推的起點。

### 3. 用公式推出後續的旋轉函數

每次從前一個結果推出下一個結果：

```text
F(k) = F(k - 1) + numSum - n * nums[n - k]
```

意思是：

- 原本每個元素的權重幾乎都整體增加 1
- 但被轉到最前面的那個元素，權重會從最大掉到 0
- 所以最後要扣掉 `n * nums[n - k]`

這樣就不用真的建立旋轉後的新陣列。

### 4. 持續更新最大值

每算出一個新的 `F(k)`，就和目前最大值比較，留下較大的答案即可。

## 演算法步驟

1. 取得陣列長度 `n`
2. 計算陣列總和 `numSum`
3. 計算 `F(0)`
4. 用 `res` 記錄目前最大值
5. 從後往前走訪會被旋轉到前面的元素
6. 依照公式更新 `F(k)`
7. 每次更新後同步更新 `res`
8. 回傳 `res`

## 複雜度分析

- 時間複雜度：`O(n)`
  只需要走訪陣列常數次
- 空間複雜度：`O(1)`
  只使用少數額外變數

## 範例演示流程

以 `nums = [4, 3, 2, 6]` 為例：

### Step 1: 計算 numSum

```text
numSum = 4 + 3 + 2 + 6 = 15
```

### Step 2: 計算 F(0)

```text
F(0) = 0 * 4 + 1 * 3 + 2 * 2 + 3 * 6
     = 0 + 3 + 4 + 18
     = 25
```

目前最大值：

```text
res = 25
```

### Step 3: 推出 F(1)

```text
F(1) = F(0) + numSum - n * nums[3]
     = 25 + 15 - 4 * 6
     = 25 + 15 - 24
     = 16
```

更新最大值：

```text
res = max(25, 16) = 25
```

### Step 4: 推出 F(2)

```text
F(2) = F(1) + numSum - n * nums[2]
     = 16 + 15 - 4 * 2
     = 16 + 15 - 8
     = 23
```

更新最大值：

```text
res = max(25, 23) = 25
```

### Step 5: 推出 F(3)

```text
F(3) = F(2) + numSum - n * nums[1]
     = 23 + 15 - 4 * 3
     = 23 + 15 - 12
     = 26
```

更新最大值：

```text
res = max(25, 26) = 26
```

### 最終答案

```text
max(F(0), F(1), F(2), F(3)) = 26
```

## 程式核心觀念總結

這題最重要的轉折點在於：

- 不要真的旋轉陣列
- 不要每次都重新計算整個加權總和
- 改用 `F(k)` 與 `F(k - 1)` 的數學關係做遞推

因此可以從原本的 `O(n^2)` 優化到 `O(n)`，這就是本題最核心的解題思路。
